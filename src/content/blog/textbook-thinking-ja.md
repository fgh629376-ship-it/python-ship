## 2. 章ごとの論理連鎖

### Ch1-2：「なぜ予測するのか」から「良し悪しの判断基準」へ

**核心的因果連鎖：**

1. 太陽光発電の出力は天候に依存する → 本質的に制御不能（変動性）
2. 系統は需給バランスを必要とする → PV出力を事前に知らなければならない → **予測が必要**
3. 予測には品質の差がある → Murphyの三次元：**一致性**（誠実さ）／**品質**（精度）／**価値**（有用性）
4. 品質が高い ≠ 価値が高い → 利用者が使えなければ価値はゼロ
5. 「一見正しいが実は的外れ」な研究を避けるには？ → 批判的思考の三つのツール：
   - **オッカムの剃刀**：必要でなければ実体を増やすな（シンプルさ優先）
   - **ヒュームの二叉路**：あらゆる主張に「証拠は何か」と問え
   - **オッカムの箒**：不都合な事実を隠していないか注意せよ

**演習問題：**

**Q1：ある論文が $R^2 = 0.95$ のみ報告し、RMSEを報告していない — 何を「隠している」可能性があるか？**

> **思考プロセス：** 太陽光発電の出力には強い日周期がある（昼間は発電、夜間はゼロ）。この周期だけで非常に高い $R^2$ が生まれる。$R^2$ は「モデルがどれだけ分散を説明するか」を測る指標 — データに明確なトレンド（朝から昼のピークへ、そして夕方にゼロへ戻る）がある場合、粗い予測でも容易に $R^2 > 0.9$ を達成できる。一方、RMSEは「予測が観測からどれだけ$\text{W/m}^2$単位で外れているか」を測り、予測スキルを直接反映する。RMSEを報告せずに $R^2$ のみを報告するのは、おそらくRMSEの見栄えが悪いためだ — 日の出・日没付近や曇天時に大きな誤差があり、$R^2$ の「見せかけの繁栄」によって隠されている可能性がある。
>
> **答え：** 真の予測誤差を「隠している」可能性がある。$R^2 = 0.95$ は太陽光予測においてほとんど識別力を持たない（パーシステンスモデルでさえこの値を達成できる）— 真の予測品質を反映するにはRMSEとMAEの両方を報告しなければならない。これは典型的な「オッカムの箒」— 有利な指標を選択的に提示している。

**Q2：XGBoostのRMSEが20層DNNより$1 \text{W/m}^2$ だけ低い — どちらを選ぶか？理由は？**

> **思考プロセス：** $1 \text{W/m}^2$ の差は統計的に有意でないかもしれない（DMテストが必要）が、たとえ有意であってもオッカムの剃刀が適用される。XGBoostは比較的シンプルなアンサンブル木モデルだ：訓練が速い、解釈可能（特徴貢献度が見える）、チューニングが容易、デプロイが簡単。20層DNNは訓練に数十倍の時間がかかり、解釈不能で、ハイパーパラメータに敏感で、過学習しやすく、デプロイにはGPUが必要だ。$1 \text{W/m}^2$ の改善はこれらのコストに見合わない。
>
> **答え：** XGBoostを選ぶ。理由：①$1 \text{W/m}^2$ の差は有意でない可能性がある（DMテスト必要）；②オッカムの剃刀 — 1%未満の改善のために10倍の複雑さを払う価値はない；③Markovics & Mayer（2022）の68モデル比較で、典型的なデータ量ではXGBoost/LightGBMが総合的に最適であることが確認されている。

### Ch3：「カオス」から「なぜ確率的予測が必要か」へ

**核心的因果連鎖：**

1. 天気は決定論的システムだ（ラプラスの悪魔）→ しかし初期条件に極めて敏感（カオス）
2. 観測には常に誤差がある → 初期条件は決して完全ではない → **決定論的予測には常に誤差がある**
3. 誤差は排除できない → 不確実性を定量化しなければならない → **確率的予測**
4. 確率的予測の完全な形式 = 予測分布 $F(x)$
   - そこから抽出できるもの：分位数 $q_\tau$、区間 $[q_{0.1}, q_{0.9}]$、平均値/中央値
5. アンサンブル予測（複数メンバー）= 経験分布 = 確率的予測の離散近似
6. 確率的予測の評価：**カリブレーション**（統計的一致性）＋**シャープネス**（狭い区間が優れる）
   - まずカリブレーションを確保 → 次にシャープネスを追求（Gneitingパラダイム）

**重要な数式：**
- CDF：$F_X(x) = P(X \leq x)$
- 分位数：$F^{-1}(\tau) = \inf\{x : F(x) > \tau\}$
- ガウス混合：$f(x) = \sum w_k \phi(x|\mu_k, \sigma_k)$

**演習問題：**

**Q3：日射量はガウス分布でモデル化できるか？理由は？**

> **思考プロセス：** ガウス分布 $N(\mu, \sigma^2)$ は $(-\infty, +\infty)$ で定義される。しかし日射量には物理的制約がある：①下限 $\geq 0$（負の日射量は存在しない）；②上限は快晴時の値の約1.5倍（雲増強効果の限界）；③夜間は正確にゼロ（点質量であり、連続分布ではない）。ガウスモデリングを使うと負の日射量という予測が生成されてしまう — 物理的に無意味だ。さらに日射量の分布は典型的に二峰性（快晴ピーク $\kappa \approx 1$ ＋曇天ピーク $\kappa \approx 0.3$）であり、単一ガウスでは捉えられない。
>
> **答え：** 否。三つの理由：①ガウスは負の無限大まで延び、負の日射量予測を生成する；②夜間には点質量 $P(GHI=0)>0$ が存在し、純粋な連続ガウスでは表現できない；③日射量は典型的に二峰性であり、単一ガウスは適合が悪い。より良い代替案：切断正規分布、ガウス混合（2〜3成分）、またはベータ分布（ただしベータも $\kappa>1$ の場合は失敗する）。

**Q4：「80%区間の実際のカバレッジが90%」 — 過分散か過小分散か？**

> **思考プロセス：** 「80%予測区間」は観測値の80%がその中に収まると期待することを意味する。実際のカバレッジが90%（公称値より高い）であれば、区間が広すぎる — 主張するよりも多くの観測値を含んでいる。区間が広すぎる = 不確実性を過大評価している = 過分散。逆に、カバレッジが80%未満であれば過小分散（過信）。
>
> **答え：** 過分散。実際のカバレッジ90% > 公称値80%、つまり予測区間が広すぎてモデルが不確実性を過大評価している。「広すぎるより良い」（狭すぎると極端な事象を見逃す）とも言えるが、過分散はシャープネスが不十分であることを意味する — 予測が伝える情報が少なく、運用上の意思決定に使いにくい。

### Ch4：太陽光発電予測 vs. 他のエネルギー予測 — 何が独自なのか？

**核心的因果連鎖：**

1. 太陽光発電 vs. 負荷/風力/価格：成熟度が最も低い（2010年代にようやく離陸）
2. 太陽光予測の**独自の優位性**：快晴期待値（他の分野にはない）
   - 快晴モデルREST2：$$B_{nc} = E_{0n} \times T_R \times T_g \times T_o \times T_n \times T_w \times T_a$$
   - 快晴インデックス：$\kappa = \text{GHI} / \text{GHI}_{\text{clear}}$ → 天文学的周期を除去 → 天気信号のみ残る
3. **GEFCom2014の重要な事実**：快晴モデルを使用した唯一のチーム = 優勝
4. Model Chain（日射量→発電量）はPV固有の変換チェーン：
   - $k_t = G_h/E_0$ → 分解 → $G_c = B_n\cos\theta + R_dD_h + \rho_gR_rG_h$ → 温度 → 発電量
5. 風力発電：$P = \frac{1}{2}C_p\rho\pi R^2 W^3$（$P \propto W^3$ → 小さな風速誤差が3乗で増幅）
6. 電力価格：負値になる可能性があり、極端なスパイク → VST変換が必要

**演習問題：**

**Q5：なぜ快晴インデックス $\kappa$ は生のGHIよりも予測モデルへの入力として優れているのか？**

> **思考プロセス：** 生のGHIには二つの成分が含まれている：①決定論的な天文学的周期（太陽位置が時間とともに変化 — 完全に計算可能）；②確率的な気象影響（雲、エアロゾルなど）。生のGHIを直接使うと、モデルはまず「すでに知っている事実」である日の出・日没を「学習」しなければならない — 既知の情報にモデル容量を無駄にすることになる。$\kappa = \text{GHI}/\text{GHI}_{\text{clear}}$ は快晴モデルで天文学的周期を除去し、純粋な気象信号を残し、$[0, \sim1.5]$ の範囲に正規化して季節性を排除する。
>
> **答え：** $\kappa$ は決定論的な天文学的周期と季節変動を除去する（快晴モデルに完全に吸収される）ため、モデルは確率的な気象駆動変動のみを学習すれば良い。同じモデル複雑度で、より難しい問題を解く → 効率が上がる。これがGEFCom2014チャンピオンのコア戦略だった — 快晴モデルを使用した唯一のチーム。

**Q6：上海とロサンゼルスのGHIは高い相関がある。これは因果関係を意味するか？**

> **思考プロセス：** 上海とロサンゼルスはともに北半球の中緯度に位置し、太陽位置の季節パターンがほぼ同一だ（夏に高く、冬に低い）。生のGHIの支配的な成分は天文学的周期 → 二都市のGHI間の高い相関は、共通の季節パターンによるものであって、一方の都市の天気がもう一方に影響しているからではない。$\kappa$（季節調整後）を使うと、二都市間の相関は劇的に低下するかゼロに近づく — 雲や気象パターンは全く異なるからだ。
>
> **答え：** 否、因果関係ではなく**擬似相関**だ。高い相関は共通の天文学的周期（ともに北半球 → 同じ季節的太陽パターン）によるものであり、因果メカニズムは存在しない。これこそが分析にGHIではなく $\kappa$ を使わなければならない理由だ — 生のGHIを使って相関を計算すると、擬似的な季節相関に誤導される。

### Ch5：理論から実践へ — データ処理の各ステップが重要

**核心的因果連鎖：**

1. 日射量成分の閉合関係：$G_h = D_h + B_n\cos Z$
2. k-インデックスシステム（正規化ツール）：
   - $\kappa = G_h/G_{hc}$（快晴インデックス、最重要）
   - $k_t = G_h/E_0$（晴天指数、より粗い）
   - $k = D_h/G_h$（散乱割合）
3. データ品質管理（QC）三段階：
   - 自動テスト：PPL/ERLの物理限界 → $-4 \leq G_h \leq 1.5E_{0n}\cos^{1.2}Z + 100$
   - 目視検査：散布図 + ヒートマップ + 快晴インデックス時系列
   - 欠損補完：衛星日射量 > 統計的補間
4. 快晴モデルの選択：REST2（最良）> McClear（中程度）> Ineichen-Perez（最もシンプル）
5. MSEスケーリング理論：$$\text{MSE}(r,x) = E[c^2] \cdot (1-\gamma_h^2) \cdot V(\kappa)$$
   - 三つの因子：予測可能性 × 変動性 × スケール
6. 時刻整合の落とし穴：集計スキームの誤り → RMSEが人工的に28%膨張！
7. 運用カルテット $(S, R_f, L, U)$ が曖昧な「予測ホライズン」に取って代わる

**演習問題：**

**Q7：なぜ時間的集計の前にQCを行わなければならないのか？**

> **思考プロセス：** 1分データに異常スパイク（例：センサー誤作動でGHI = $5000 \text{W/m}^2$）が含まれているとする。まず時間平均に集計すると、このスパイクは60個の正常値で薄まり — 時間平均値はわずか数十$\text{W/m}^2$高くなるだけで、QC閾値内に収まり検出を免れる。しかし1分解像度でQCを適用すれば、スパイクはPPL上限（$1.5E_{0n}\cos^{1.2}Z + 100$）をはるかに超え、即座にフラグが立てられ除去される。集計後のQC = 情報損失 → 見逃し増加。
>
> **答え：** 時間的集計（平均化）は**外れ値を薄める**ため、正常範囲内に隠れてQC検出を免れる。1分解像度の極端なスパイクは、時間平均では59個の正常値に洗い流される。したがって、QCは集計の前に行わなければならない — 最高解像度で異常を検出し、その後クリーンなデータで時間平均を行う。

**Q8：REST2とIneichen-Perezの性能差はどれくらいか？選択の基準は？**

> **思考プロセス：** Ch5.4.5は7つのSURFRADステーションの4年データで三つのモデルを比較した。最も重要な発見は：**快晴モデル間の差は、ステーション間の差よりはるかに小さい**。つまり、観測地点の気候特性（砂漠 vs. 海洋性）が予測難度に与える影響は、快晴モデルの選択の影響よりはるかに大きい。REST2は75モデル中で最高ランクだが、中程度のランクのIneichen-Perezで十分だ。
>
> **答え：** 差は小さい — ステーション間差 >> モデル間差。選択基準：①最高精度 + 再解析データあり → REST2（9入力パラメータ、MERRA-2必要）；②最も実装が容易 + 時間範囲制限なし → Ineichen-Perez（Linke濁度係数のみ必要、数行のコード）；③オンライン製品が必要 → McClear（SoDaウェブサイトで無料、ただしオープンソースではない）。実際の差は予想よりはるかに小さく、オッカムの剃刀はよりシンプルな選択肢を支持する。

### Ch6：データは予測の天井

**核心的因果連鎖：**

1. 三つのデータソースにはそれぞれ長所がある：
   - **地上観測**：最も正確（不確実性5%）、しかし極めて疎（中国では約100局のみ）
   - **衛星日射量**：広いカバレッジ（全球低・中緯度）、中程度の精度、5分/0.5km
   - **NWP/再解析**：時空間的に最も完全 + 自己整合的な変数、しかし最低精度
2. 代表的なデータセットの三つの要件：**カバレッジ**（時空間・気候・空況）／**一致性**（スケール整合）／**アクセス可能性**（再現性）
3. **再現不可能 = 疑似科学** → 公開データセットを使用しなければならない
4. 主要データベース：BSRN（世界70局以上）、SURFRAD（米国7局、最高品質）、NSRDB（全球、快晴値を含む）

**演習問題：**

**Q9：NWPグリッド点は5〜50km²の平均を表し、地上観測は単一点だ。直接比較すると何の問題が生じるか？**

> **思考プロセス：** 雲は空間変動性が高い — 積雲の幅はわずか500mかもしれない。NWPグリッド点は10km$\times 10$kmの領域の平均日射量を表し、地上ステーションは**単一点**を測定する。雲がたまたま地上ステーションを覆っているが周囲は覆っていない場合、地上GHIは低くなりNWPグリッド平均GHIは妥当なまま — この「誤差」はNWPの予測精度が悪いからではなく、空間スケールの不一致による代表誤差だ。
>
> **答え：** **代表誤差**が生じる。単一点の地上観測にはサブグリッドスケールの変動（特に雲による急速な変動）が含まれており、NWPではこれを解像できない。この誤差は予測手法の問題ではなく、検証アプローチの欠陥だ。解決策：①NWPグリッドスケールに合わせるため複数ステーション平均を使う；②すでに面積平均である衛星日射量を検証に使う；③評価においてシステム内の代表誤差の寄与を考慮する。

**Q10：なぜERA5再解析は予測検証の地上観測の代替として直接使えないのか？**

> **思考プロセス：** 再解析は「固定された」モデルを使って過去データを再分析するものであり、その日射量出力はそれ自体がモデル産物だ — 独自の系統バイアスと誤差特性を持つ。あるモデル産物を使って別のモデルを検証することは、「真値」の代わりに「近似的な真値」を使うことになる。ERA5の日射量精度（〜31km解像度）は高品質な地上観測（不確実性5%）より明らかに劣る。ERA5で検証すると：①特定の誤差が過小評価される可能性がある（両モデルが同じバイアス源を共有する場合）；②擬似的な結論が導かれる可能性がある。
>
> **答え：** ERA5日射量はそれ自体がモデル産物であり、実際の観測ではないため。独自の系統バイアス（特に雲シミュレーションが不正確）と比較的粗い解像度（31km）を持つ。あるモデルを別のモデルで検証する → 誤差を系統的に過小評価する可能性がある。地上観測はカバレッジが限られているものの、「真値」に最も近い。妥協案：高品質な衛星日射量（例：NSRDB）が広域検証の「準真値」として使える。

### Ch7：三つの基本予測手法の物理的基礎

**核心的因果連鎖：**

1. **NWP（翌日予測の唯一の選択）**：
   - 7つの原始方程式：$\frac{D\theta}{Dt} = \frac{\theta}{T}\frac{Q}{c_p}$、$p = \rho RT$
   - 6つのパラメタリゼーションスキーム：放射 / 雲微物理 / 対流 / PBL / 地表面 / エアロゾル
   - ボトルネックは放射スキームではなく**雲微物理**
   - CFL安定性：$\Delta t \leq \Delta x / c$ → グリッドが細かい = 計算コスト増大
   - データ同化（4D-Var）：観測値を初期条件に組み込む
2. **衛星（日中補完、<4h）**：
   - 雲移動ベクトル（CMV）→ 凍結場仮定 → 15〜30分後に失効
   - 三つの推定タイプ：物理的（FARMS）/ 半経験的（Heliosat）/ ML
3. **地上統計（超短期、<1h）**：
   - パーシステンス参照：$\kappa(t+h) = \kappa(t)$
   - CLIPER：$\rho_t = \gamma_h \kappa_{t-h} + (1-\gamma_h)E(\kappa)$（数学的最適解）
   - 時空間手法（STAR/Kriging/GNN）はやや優れるがまだ限界あり

**演習問題：**

**Q11：「純粋なデータ駆動手法がNWPを翌日予測で置き換えられる」はなぜ幻想なのか？**

> **思考プロセス：** 翌日予測 = 24〜36時間先。このホライズンでは、快晴インデックス $\kappa$ の時系列の自己相関 $\gamma_h$ はすでにゼロに近い — つまり**現在の $\kappa$ は24時間後の $\kappa$ についてほとんど予測力を持たない**。純粋なデータ駆動手法（LSTM/Transformer）は過去データからの情報しか抽出できないが、情報理論は言う：「情報を含まない入力から情報を抽出することは不可能だ」。NWPが翌日予測を行えるのは、大気の原始方程式を解くためだ — 現在の大気状態から物理的に未来の状態を外挿し、過去データには存在しない新しい情報を導入する。
>
> **答え：** $\gamma_h \to 0$（自己相関が24hで消失）のため、過去データには24時間後の天気に関する**情報が含まれていない**。データ駆動手法はデータ内の情報しか採掘できないのに対し、NWPは物理方程式（大気ダイナミクスの外挿）を通じて新しい情報を**生成する**。これはモデル複雑度の問題ではなく情報理論からのハード制約だ — 100層のTransformerでもこれを破れない。

**Q12：なぜLSTMは4時間を超える予測で失敗するのか？**

> **思考プロセス：** LSTMのコア能力は長期的な逐次パターンを記憶することだ。しかし快晴インデックス $\kappa$ の自己相関 $\gamma_h$ はリードタイム $h$ とともに急速に減衰する。$h < 1h$ では $\gamma_h$ は比較的高く、LSTMは最近の $\kappa$ 値を活用して効果的な予測ができる。しかし $h > 4h$ を超えると $\gamma_h \approx 0$ となり — 現在の $\kappa$ と未来の $\kappa$ は統計的に独立であり、過去の情報は予測に役立たなくなる。LSTMの「記憶」は無価値になる — 記憶していることが未来と無関係だから。
>
> **答え：** $\kappa$ の自己相関 $\gamma_h$ が4hを超えるとゼロに近づくため。LSTMの記憶がどれほど強力でも、統計的に独立な入力から予測情報を抽出できない。この時点で唯一の新しい情報源はNWP（統計的相関ではなく物理方程式を通じて）だ。これはCh4の「手法-ホライズン」チャートで統計的手法が4hを超えるとNWPに完全に追い越される理由も説明している。

### Ch8：後処理 — 生の予測から最終製品へ

**核心的因果連鎖：**

後処理 = 予測変数を変えずに（GHIはGHIのまま）、品質のみを向上させる。入力/出力によって四つの象限に分類：

**D2D（決定論的→決定論的）：**
- 回帰：$\hat{y} = \beta_0 + \beta_1 x$（ドメイン知識を取り込む > ブラックボックスML）
- カルマンフィルタリング：$Y_t = \mathbf{x}_t^T\beta_t + \varepsilon_t$（オンライン更新、運用追跡）
- ダウンスケーリング：確率化を追求（単一ダウンスケールシーケンスのランプイベントは必然的にずれる）

**P2D（確率的→決定論的）：**
- 適切な統計汎関数を選ぶ：MSE → 平均値、MAE → 中央値、Pinball → $\tau$-分位数
- 予測の組み合わせ：$\text{MSE}_{\text{combo}} = \frac{1}{m}\sum\text{MSE}_j - \text{多様性項}$
- **多様性が鍵**：物理 + データ駆動の混合 > 同質MLモデルの集合

**D2P（決定論的→確率的）：**
- アナログアンサンブルAnEn：天気パターンは繰り返す → 過去の類似状況を探す
- ドレッシング法：$N(0, \sigma_t^2)$（過去の誤差 → 不確実性推定）
- 分位数回帰 / GAMLSS：$D(\mu, \sigma, \nu, \tau)$

**P2P（確率的→確率的）：**
- EMOS：平均 $= w_0 + \sum w_j x_j$、分散 $= \beta_0 + \beta_1 S_t^2$
- BMA：$g_t(x) = \sum w_j f_{tj}(x)$
- 核心原則：**確率的→決定論的 > 直接決定論的**

**演習問題：**

**Q13：なぜ等重み平均を打ち負かすことが難しいのか？**

> **思考プロセス：** 予測組み合わせのMSEは次のように分解される：$\text{MSE}_{\text{combo}} = \frac{1}{m}\sum\text{MSE}_j - \text{多様性項}$。多様性項 $= E[(X_j - X_k)^2] \geq 0$ は非負 — つまり組み合わせのMSEは**メンバーの平均MSEを超えないことが保証される**。等重み平均はこの保証を自動的に得る — 重みを推定するための訓練データも不要だ。不均等な重みの使用には $w_j$ の推定が必要であり、推定自体がノイズを導入する — 特に小サンプルでは、重み推定の不安定さから不均等な重みが悪い結果を出す可能性がある。Armstrongのルールはうまく要約している：「不均等な重みを支持する強い証拠がない限り、等重みを使え。」
>
> **答え：** MSE分解の多様性項が $\geq 0$（数学的保証）であるため、等重み組み合わせの誤差はメンバー平均誤差を超えないことが保証される。不均等な重みには重みの推定が必要であり、推定ノイズが理論的優位性を打ち消す可能性がある。等重み = 推定リスクゼロ + 多様性配当。これが68のMLモデルの等重み平均を個別モデルや不均等重みスキームで打ち負かすことが難しい理由だ。

**Q14：カルマンフィルタリングは1ステップ先のみ行う — 複数ステップの予測はどうするか？**

> **思考プロセス：** カルマンフィルタリングの再帰構造は：観測値到着 → 状態更新 → 次ステップ予測。自然に $t+1$ のみを予測する。$t+2, t+3, \ldots, t+h$ を予測するには、1ステップ予測を入力として再帰させることはできない（誤差が急速に蓄積する）。Yangら（2017c）が見つけた解決策は：各リードタイム $h$ に対して独立したカルマンフィルタを訓練すること。$h=1$ に一つのフィルタ、$h=2$ に一つ、$\ldots$ それぞれが自分のリードタイムのバイアス特性を独立して追跡する。
>
> **答え：** 各リードタイム $h$ に対して独立したカルマンフィルタを構築する（$h$個の並列フィルタ）。各フィルタは自分のリードタイムのみを担当し、バイアスを独立して追跡する。これにより複数ステップ再帰の誤差蓄積問題を回避できる。Yangら（2017c）が太陽光発電予測においてこの問題と解決策を最初に明らかにした。

### Ch9：決定論的検証 — 「指標で勝つ」≠「予測で勝つ」

**核心的因果連鎖：**

1. 単一指標の最適化は**意味がない**：
   - 初心者はMBEで勝つ、楽観主義者はMAEで勝つ、統計学者はRMSEで勝つ
   - 同じデータセットで、三つの指標が三つの異なる「最良」モデルを選ぶ
2. **なぜ？** → 異なる指標は異なる種類の誤差にペナルティを与える
   - RMSEは大きな誤差にペナルティ（二乗）、MAEはすべての誤差を等しく重み付け、MBEは正負が打ち消し合う
3. スキルスコア：$S^* = 1 - S_{\text{fcst}} / S_{\text{ref}}$
   - パーシステンスではなくCLIPERを参照として使う：$\gamma_h \kappa_{t-h} + (1-\gamma_h)E(\kappa)$
   - CLIPERはパーシステンスと気候値の両方より厳密に優れている（数学的に証明済み）
   - **CLIPERに勝てないモデルには存在する理由がない**
4. Murphy-Winklerの分布志向：f-o結合分布の三つのスライスを検討
5. **品質 vs. 価値**：9%の品質差 → 5倍のペナルティ差（非線形マッピング）

**演習問題：**

**Q15：モデルAのMAE = 50、モデルBのMAE = 52 — Aの方が優れていると言えるか？**

> **思考プロセス：** $2 \text{W/m}^2$の差はランダムな変動から来ている可能性がある。あるテストセットではたまたまAがBを2だけ上回るが、別のテストセットではBがAを抜くかもしれない。統計は「真の差異」と「ランダムノイズ」を区別することを求める。Diebold-Mariano（DM）テストはまさにこれを行う：各時刻ステップで二つのモデルの損失差 $d_t = L(e_{A,t}) - L(e_{B,t})$ を計算し、$H_0: E[d_t] = 0$ をテストする。DM統計量は近似的に $N(0,1)$ であり、有意差を主張するにはp値 < 0.05が必要だ。
>
> **答え：** Aの方が優れているとは直接言えない。**Diebold-MarianoテスT**が必要だ：$H_0: E[d_t] = 0$、$d_t = |e_{A,t}| - |e_{B,t}|$（MAEの絶対誤差損失）。p値が十分に小さい場合（例：< 0.05）のみ、AがBより統計的に有意に優れていると言える。太陽光予測の論文ではこのテストがほとんど行われない — 多くは単に数値を比較するのみで、統計的に不適切だ。

**Q16：なぜ「MAPE最適予測が高いペナルティを招く」のか？**

> **思考プロセス：** MAPEはパーセント誤差にペナルティを与える — 日射量がゼロに近い場合（日の出・日没周辺）、小さな絶対誤差（例：$5 \text{W/m}^2$）でも分母が小さい（例：$10 \text{W/m}^2$）ため巨大なMAPEが生じ、MAPEが50%になる。MAPE最適モデルはこれらの端点時刻を「最適化」し、低日射量での予測を非常に正確にする — しかしこれらの時刻は発電がほとんどなく、系統的な価値はほぼゼロだ。一方、真昼の大きな絶対誤差（実際$50 \text{W/m}^2$に対して$500 \text{W/m}^2$の予測；MAPEはわずか10%）は無視される可能性があり、それらの誤差が不均衡ペナルティの主な原因となる。
>
> **答え：** MAPEが低日射量時（日の出・日没）の小さな誤差に過大な重みを与えるため、モデルは経済的価値が最も低い時刻を「最適化」し、高日射量時の大きな絶対誤差を無視するようになる — そして系統ペナルティは通常絶対偏差に比例する。これがCh9.5の核心的発見だ：**品質（良い指標）と価値（経済的便益）には非線形な関係がある** — 9%の品質差が5倍のペナルティ差を引き起こすことがある。

### Ch10：確率的検証 — 「正しい不確実性」は「正確な点値」より重要

**核心的因果連鎖：**

1. 系統は予備容量を必要とする → 予備容量は不確実性の大きさに依存する → **確率的予測の核心的価値**
2. Gneitingパラダイム：**カリブレーションを維持しながらシャープネスを最大化する**
   - カリブレーション = PITが一様分布に従う：$F_t(y_t) \sim U(0,1)$
   - シャープネス = 予測区間はできるだけ狭いべき
   - 落とし穴：気候値は完全にカリブレーションされているがシャープネスはゼロ → 無価値
3. 四つの厳密に適切なスコアリングルール：
   - **CRPS**：$$\text{crps}(F,y) = \int_{-\infty}^{\infty}[F(x) - \mathbb{1}(x \geq y)]^2\,dx$$
   - CRPS = MAEの確率的一般化 = すべての閾値にわたるBrierスコアの積分
   - **IGN**：$-\log f(y)$（局所スコア、観測点でのPDF値のみ見る）
4. **一貫性原則**：評価に使うものを訓練にも使う
   - CRPS評価 → CRPS訓練 ✅
   - IGN評価 → 尤度訓練 ✅
   - 混合 → 次善 ❌

**演習問題：**

**Q17：CRPSの単位は何か？MAEと同じか？**

> **思考プロセス：** CRPSは $\int[F(x) - \mathbb{1}(x \geq y)]^2 dx$ として定義される。被積分関数 $[F(x) - \mathbb{1}]^2$ は無次元（確率差の二乗）だが、積分変数 $dx$ は予測変数と同じ単位（$\text{W/m}^2$）を持つ。したがってCRPSの単位 = 予測変数の単位。予測分布が点予測 $\delta(\hat{y})$ に縮退すると、$\text{CRPS} = |y - \hat{y}| = \text{MAE}$。両者は同じ単位を持ち、CRPSとMAEを数値的に直接比較できる。
>
> **答え：** CRPSは予測変数（$\text{W/m}^2$）と同じ単位を持ち、MAEと同じだ。これはCRPSの重要な優位点 — 確率的スコアでありながら物理的な解釈可能性を保持している。確率的予測が点予測に縮退するとき、CRPSはMAEと正確に等しくなるので、両者を数値的に直接比較できる。

**Q18：「厳密に適切」とはどういう意味か？スコアリングルールが厳密に適切でない場合は何が起きるか？**

> **思考プロセス：** 「適切」とは：予測者が**真の信念**に従って予測するとき期待スコアが最適化される — 予測を歪める動機がないことを意味する。「厳密」はさらに進んで：真の信念に従って予測した場合**のみ**最適が達成され、いかなる逸脱も悪化させる。スコアリングルールが厳密に適切でない場合、予測者は「戦略的ヘッジ」でスコアを改善できる — 例えば雨が降ることを知っていながら期待損失を減らすために80%の代わりに60%を報告する。これは検証結果を汚染する：高スコア = 良い予測ではなく、単に「ゲーム操作」が巧みなことを意味するかもしれない。
>
> **答え：** 「厳密に適切」は**真の信念に従った予測のみが期待スコアを最適化する** — いかなる戦略的逸脱も悪化させる → ゲーム操作を抑止する。スコアリングルールが厳密に適切でない場合、予測者は真の信念を表さない予測（ヘッジ）を出す動機を持ち、検証結果を汚染する — 高スコア ≠ 良い予測。これがCRPS、IGN、Brierスコアが推奨される理由だ：すべて厳密に適切だ。

### Ch11：日射量→発電量 — Model Chainを通じた誤差伝播

**核心的因果連鎖：**

1. 系統が必要とするのは**発電量**予測であり、日射量予測ではない
2. 三つの変換手法：
   - **直接回帰**：NWP変数 → ML → 発電量（過去データが必要）
   - **物理的Model Chain**：分解 → 変換 → 反射 → 温度 → PVモデル → 損失
   - **ハイブリッド**：物理フロントエンド + MLバックエンド（通常最適）
3. 特徴量重要度ランキング：**快晴情報 > 特徴量エンジニアリング > 確率的/アンサンブル > モデル選択**
4. 分解モデルの進化：ERBS(1982) → BRL(2010) → ENGERER2(2015) → YANG4(2021、現在最良)
5. **エンドツーエンド最適化の必要性**：
   - 分解モデルAが散乱を5%過大推定 → 変換モデルBが散乱に敏感 → 全体誤差が増幅
   - 最良の個別モデルの組み合わせ ≠ 最良のModel Chain
6. 温度効果：$T_{\text{cell}} = T_{\text{amb}} + \frac{\text{NOCT}-20}{800} \times G_c$
   - $\gamma \approx -0.4\%/°C$ → 砂漠の40°Cは25°C STCと比較して約6%の発電損失をもたらす

**演習問題：**

**Q19：新しい発電所で過去データがない場合 — どの手法が使えるか？理由は？**

> **思考プロセス：** 三つの手法はデータ要件が異なる。直接回帰には「入力-発電量」の過去訓練データが必要 — 新しい発電所にはそれがない。物理的Model Chainは発電所の設計パラメータ（パネルモデル、傾斜角、方位角、インバータ仕様）と日射量予測のみを必要とし — 過去の発電量データは不要だ。ハイブリッド手法のMLバックエンドも訓練データを必要とする。したがって過去データがゼロの場合、使えるのは純粋な物理的Model Chainのみだ。
>
> **答え：** **物理的Model Chain**のみが使える。必要なのは発電所の設計パラメータ（パネルパラメータ、設置角度など）と日射量予測入力のみであり、訓練のための過去発電量データは不要だ。これが二段階予測アプローチの「データスケーラビリティ」優位点 — 新しい発電所はデータ蓄積を待たずに即座に予測を生成できる。十分な過去データが蓄積されたら、さらなる最適化のためにハイブリッド手法に切り替えられる。

**Q20：なぜ「快晴情報 > モデル選択」なのか？Ch4のどの結論に呼応しているか？**

> **思考プロセス：** Ch11.1の特徴量重要度ランキングは、モデルに生のGHIではなく快晴インデックス $\kappa$ を提供することが、より複雑なモデルに切り替えるよりも大きな改善をもたらすことを示している。理由：$\kappa$ は既知の天文学的信号を除去し、モデルが気象信号に集中できるようにする — これは情報レベルでの改善だ。モデルを切り替えることは単に適合能力を微調整するだけ — 同じ情報入力で、異なるモデルが抽出できる情報量の差はわずかだ。
>
> **答え：** 快晴正規化は**情報レベル**での改善（既知の信号を除去 → 有効情報密度が上がる）だが、モデル選択は単に**適合能力**を微調整するだけだ。これはCh4.6.2に直接呼応している：「LSTMに快晴インデックスを与える >> LSTMに日の出・日没を自分で学習させる」、そしてGEFCom2014の結論にも呼応する：快晴モデルを使用した唯一のチーム = チャンピオン。**ドメイン知識の価値 > モデル複雑度の価値。**

### Ch12：階層的予測とFirm Power — 究極のクローズドループ

**核心的因果連鎖：**

1. 電力システムは本質的に階層的だ：連系→送電ゾーン→発電所→インバータ
2. 各レベルで独立した予測 → 合計が整合しない → 給電混乱
3. 階層的調整：$$\tilde{\mathbf{y}} = \mathbf{S} \cdot \mathbf{P} \cdot \hat{\mathbf{y}}$$
   - 5手法：BU / OLS / WLS / HLS / **MinT-shrink**（最適）
   - 理論的保証：調整後、すべてのレベルの予測 ≥ 独立予測
4. **Firm Power（究極目標）** = 予測 + 蓄電池 + 給電 → 給電可能な太陽エネルギー
   - Firm Forecasting（予測誤差の排除）→ コスト〜2倍
   - Firm Generation（変動性の排除）→ コスト〜3倍
5. 四つのイネーブラー：
   - **蓄電池**：核心だが高価 → 容量最適化が必要
   - **地理的平滑化**：分散立地 → 自然の「予測組み合わせ」
   - **負荷整形**：需要を供給に適応させる（需要側革命）
   - **過剰建設+出力制限**：直感に反するが必要（すべての実証研究で一致）
6. **クローズドループ**：より良い予測 → 必要な蓄電池が少ない → 固定費が低い → **予測研究には直接的な経済的価値がある**

**演習問題：**

**Q21：なぜ「過剰建設+出力制限」が最低コストのFirm Powerの必要な要素なのか？**

> **思考プロセス：** 限界コストを考えよう。蓄電池の目的はPV出力が不十分な期間を補うことだ。最後の5%の極端なケース（連続した曇天日）をカバーするには大量の追加蓄電池が必要だが、これらは年に数回しか使われず、限界利用率が極めて低い。PV容量を1.5倍に過剰建設すると、晴れた真昼の余剰発電は出力制限（蓄電せず）で対処でき、ほとんどの時間は追加容量が蓄電池の必要量を減らす。10%の発電を制限することで30%の蓄電池削減を得る — 制限の限界コストは等価な蓄電池の限界コストよりはるかに低い。
>
> **答え：** **制限の限界コスト < 等価な蓄電池の限界コスト**だから。過剰建設によりPVはほとんどの時間で余剰出力を持ち、蓄電池への依存を減らす。一部の発電が制限されても（無駄に見えるが）、蓄電池投資の節約は制限損失をはるかに超える。すべての実証研究（CAISOケースを含む）で一貫して示されている：最適なFirm Power解は過剰建設比率〜1.13〜1.54倍であり、対応する制限率は総システムコスト最小化のための必要なコストだ。

**Q22：Ch1からCh12まで、完全な因果連鎖を5ステップ以内で書け。**

> **思考プロセス：** 全12章を5つのコアステップに蒸留し、各ステップが前のステップの論理的結果となるように：
>
> **答え：**
> 1. **PVの変動性 → 予測が必要** (Ch1-2：問題定義)
> 2. **不確実性は排除できない → 確率的予測 + 快晴正規化** (Ch3-5：理論・データの基礎)
> 3. **NWP + 後処理 → 予測の生成と改善** (Ch6-8：方法論)

> 5. **Model Chain + 階層的調整 + 蓄電池 → 給電可能な太陽エネルギー** (Ch11-12：究極の応用)
>
> 一文で：**変動性 → 確率的予測 → NWP + 後処理 → 厳密な検証 → Firm Power**。
