## 🔗 Ch1-2 逻辑链：从"为什么预测"到"怎么判断好坏"

**推演起点**：光伏有间歇性

```
光伏有间歇性
  → 电网发用电平衡被打破
    → 需要预测来恢复平衡
      → 预测精度直接影响经济成本（1% ≈ 数百万美元/GW）
        → 预测研究有实际价值

但"有价值"不等于"做得好"
  → 需要定义什么是"好的预测"（Ch1: Murphy 三维度）
    → 一致性：预测者是否诚实？
    → 质量：预测与观测是否吻合？
    → 价值：预测是否帮助了决策？
      → 三者可以冲突！（高质量但用户不会用 = 低价值）

如何确保研究质量？
  → 需要批判性思维工具（Ch2）
    → 奥卡姆剃刀：复杂模型必须用精度提升来证明
    → 奥卡姆扫帚：警惕选择性报告
    → 可证伪性：不能复现的研究 = 没有价值
```

**训练问题**：
1. 如果一个模型 RMSE 降了 0.5% 但复杂度增加了 10 倍，应该用它吗？
   → 奥卡姆剃刀说不。除非 0.5% 通过了 Diebold-Mariano 检验证明显著。
2. 如果概率预测比确定性预测更好，为什么工业界不用？
   → 因为 Murphy 的"价值"维度由**用户**定义，不由预测者定义。用户不会用 = 没有价值。

## 🔗 Ch3 逻辑链：从"混沌"到"为什么必须做概率预测"

**推演起点**：天气是混沌系统

```
天气是混沌系统（确定性但对初始条件敏感）
  → 初始条件永远不完美（观测有误差）
    → 预测误差会指数增长
      → 单值预测（确定性预测）必然不完美
        → 需要量化不确定性
          → 概率预测

概率预测需要输出什么？
  → 预测分布（CDF/PDF）
    → 四种形式：参数型/半参数型/非参数型/经验型
      → 辐照度有物理上下界 → Gaussian 不适合
      → 辐照度分布双峰 → 混合高斯适合

如何评价概率预测的好坏？
  → Gneiting (2007)：校准约束下最大化锐度
    → 校准 = 说80%就真的80%
    → 锐度 = 区间越窄越好
    → 但两者存在张力：区间太窄 → 校准失败

如何生成多个预测来量化不确定性？
  → 集成预测
    → 三种来源：数据/过程/参数 不确定性
    → 三种ML策略：
      → Bagging（降方差）→ Random Forest
      → Boosting（降偏差）→ XGBoost
      → Stacking（最优组合）→ 超级学习器
```

**训练问题**：
1. 为什么 Bagging 能降低方差但不能降低偏差？
   → 因为 Bagging 是平均多个模型。如果每个模型都有相同的系统性偏差（高偏差），平均后偏差不变。但各模型的随机波动（方差）会被平均消除。
2. XGBoost 为什么在 68 模型比较中胜出？
   → Boosting 降偏差 + 树模型处理非线性 + 正则化防过拟合 = 在中等数据量下的最优平衡。深度学习需要更多数据才能发挥优势。
3. 从混沌理论出发，为什么NWP集成（50个成员）比单次预测好？
   → 混沌系统对初始条件敏感 → 扰动初始条件得到50条轨迹 → 轨迹的分散程度反映不确定性 → 集成均值比任何单条轨迹更稳健。

## 🔗 Ch4 逻辑链：从"光伏的独特性"到"最佳预测路径"

**推演起点**：光伏预测有什么是其他领域没有的？

```
光伏辐照度 = 晴空分量 × 天气调制
  → 晴空分量可以精确计算（天文+大气物理）
    → 这是其他能源预测没有的优势！
      → 用晴空模型去除季节性 → 得到晴空指数 κ
        → κ 只包含天气信号（云的影响）
          → 模型只需学习天气变化，不用学"太阳会升起"
            → GEFCom2014 唯一用晴空模型的团队 = 冠军

去除季节性后的 κ 有什么特征？
  → 双峰分布（晴天≈1.0，阴天≈0.3-0.5）
    → 不能用 Gaussian/Beta → 用混合高斯
  → 可以超过 1.0（云增强，上界≈1.5）
    → QC 上界不能设为晴空值
  → 时空相关性完全来自移动云
    → 云的运动速度决定了预测的时空关联范围
      → 相机(<30min) < 卫星(<4h) < NWP(>4h)
        → 电网需要 >4h → NWP 是唯一选择

从辐照度预测到功率预测：
  → 不是简单乘法 → 需要 Model Chain
    → 分离 → 转置 → 温度 → 功率
      → 每步有误差 → 误差会传播
        → 最优个体模型组合 ≠ 最优链
          → 必须整体优化

综合所有跨域教训：
  → 负荷预测：两阶段法（主特征+残差）
    → 光伏版 = 晴空模型（主特征）+ ML（κ残差）
  → 风力预测：>3h 必须 NWP
    → 光伏同理
  → 电价预测：VST/分解/窗口平均
    → 光伏可借鉴
```

**最终推导——光伏点预测的最佳路径**：

```
最佳路径 = 物理 + 数据驱动的融合

Step 1: NWP 提供日前天气预报（不可替代）
Step 2: 晴空模型计算 GHI_clear（去季节性）
Step 3: κ = GHI_nwp / GHI_clear（归一化）
Step 4: ML 后处理 κ（修正 NWP 系统偏差，如 MOS）
Step 5: 还原 GHI = κ_corrected × GHI_clear
Step 6: Model Chain → 功率
Step 7: 概率化（分位数回归 or 集成）

为什么这条路径是最优的？
  → Step 1: Ch4.5.4 证明了 NWP 的不可替代性
  → Step 2-3: Ch4.5.1 证明了晴空模型是最大武器
  → Step 4: Ch4.2 的两阶段法 + Ch4.4.3 的 MOS
  → Step 5-6: Ch4.5.5 的 Model Chain
  → Step 7: Ch3 证明了概率预测的必要性
```

**训练问题**：
1. 为什么不能跳过 Step 2-3 直接用 ML 预测 GHI？
   → 因为 ML 会花大量容量去学习太阳升落的周期——这是已知的物理知识，不该浪费模型容量。晴空模型免费给你这些信息。
2. 为什么 Step 6 不能用简单的 P = GHI × 效率？
   → 因为功率取决于 POA（面板平面辐照度），不是 GHI。还有温度效应、反射损失、逆变器效率。Model Chain 的每一步都有物理意义。
3. 如果数据只有 GHI 和功率（没有 NWP），还能做什么？
   → 只能做 <4h 的短期预测。用 κ 的自回归（持续性 + 滞后项），配合 Smart Persistence 做 baseline。超过 4h 就不行了。

## 🔗 Ch5 逻辑链：从"理论"到"实操"——数据处理的每一步都有讲究

**推演起点**：Ch1-4 告诉我们"做什么"和"为什么"，Ch5 开始告诉"怎么做"

```
理论知识已经充分（Ch1-4）
  → 开始实操时的第一个问题：数据质量
    → 脏数据 + 完美模型 = 垃圾结果
      → QC 是所有工作的第一步（Ch5.3）
        → 但 QC 不是简单的删异常值
          → 需要物理约束（PPL/ERL 测试用到了 E0n 和 cos(Z)）
          → 需要多分量交叉验证（三分量闭合测试）
          → 需要人工目视（自动测试会漏掉/错删数据）
          → QC 后必然有缺失 → 需要填充（首选卫星辐照度）

QC 完成后的下一步：归一化
  → Ch4 已经证明晴空模型是最大武器
    → 但具体用哪个晴空模型？（Ch5.4）
      → REST2（最准但复杂）vs McClear（中等）vs Ineichen-Perez（简单）
        → 结论：选择影响远小于地理位置影响
          → 奥卡姆剃刀(Ch2.5)：简单模型够用就不用复杂的
        → 但即使最好的模型也无法让 κ 完全平稳
          → MSE 缩放只是近似 → 局部平稳

归一化后的陷阱：时间对齐（Ch5.5）
  → NWP 是瞬时值还是平均值？→ 用错聚合方案 → 28% 虚假误差！
  → 日出/日落小时的晴空辐照度为零？→ 用1分钟分辨率先算再聚合
  → 看似微小的数据处理细节 → 可以毁掉整个验证结果

最后：运营化（Ch5.6）
  → 学术研究的"1步提前预测"在运营中毫无意义
    → CAISO 需要 (S=65min, Rf=5min, L=7.5min, U=5min) 的预测
    → 你的 h=1 小时预测满足不了这个要求
      → 必须用四元组(S,Rf,L,U)精确描述预测规格
```

**训练问题**：
1. 为什么 QC 必须在时间平均之前做？
   → 因为1分钟数据的云增强（GHI > 1.5×晴空值）在平均到小时后会消失。如果先平均再 QC，用小时数据的上界检验会删掉合理的数据点，或者漏掉实际有问题的分钟级数据。

2. 为什么晴空模型的选择对预测性能影响不大？
   → MSE 分解为三项：(1-γ²) × V(κ) × E[c²]。三个晴空模型产生的 γ 和 V(κ) 差异很小（Table 5.3），而站点之间的差异（反映气候）远大于模型之间的差异。这说明预测性能主要由天气/气候决定，而不是归一化精度。

3. 为什么"1步提前小时预测"在运营中没有价值？
   → 因为运营需要：①提前提交（Lead time > 0）②指定分辨率（可能是5分钟不是1小时）③连续更新（Update rate）④覆盖一定时间窗（Span）。学术论文的 h=1 小时预测缺少这些运营要素，无法直接集成到调度系统。

## 🔗 Ch6 逻辑链：数据决定了预测的天花板

**推演起点**：模型再好，数据不行也白搭

```
预测模型需要数据输入
  → 数据有三种来源，各有优缺：
    ┌──────────┬──────────┬──────────┬──────────┐
    │          │ 地面观测  │ 卫星反演  │ NWP/再分析│
    ├──────────┼──────────┼──────────┼──────────┤
    │ 精度     │ 最高      │ 中等      │ 最低      │
    │ 空间覆盖 │ 点(极稀疏)│ 网格(全球)│ 网格(全球)│
    │ 时间覆盖 │ 站点建立后│ ~20年     │ 40-80年   │
    │ 变量丰富 │ 辐照度为主│ 辐照度+云 │ 数百变量   │
    │ 成本     │ 20万美元/站│ 免费      │ 免费/收费 │
    └──────────┴──────────┴──────────┴──────────┘

  → 三种数据互补，不是互相替代
    → 地面观测：验证的"真值"（虽然本身有5%不确定度）
    → 卫星辐照度：NWP 后处理的空间参考（区域预测的唯一选择）
    → NWP预测：>4h 预测的唯一来源（Ch4.5.4 已证明）
    → NWP再分析：晴空模型(REST2)的大气参数来源

数据代表性的三要素：
  ① 覆盖 → 多气候区+至少一年+包含 ramp 事件
    → 只在沙漠测试的模型不能声称"通用"
    → KGC分类不够 → 需要KGPV光伏专用分类
  ② 一致性 → 数据尺度必须匹配预测方法
    → 传统"相机→分钟/卫星→小时/NWP→天"关联已过时
    → 新一代卫星(0.5km/5min)和高分辨率NWP(200m/2min)打破旧框架
    → 尺度不匹配是最常被忽视的验证误差来源
  ③ 可获取性 → 不可复现 = 伪科学（Karl Popper）
    → 公开数据集质量更高+处理工具更完善+结果可比较
    → 私有数据必须用Rapoport规则证明公开数据不够用

数据选择的决策树：
  需要日前预测？→ ECMWF HRES/ENS（最准但收费）或 GFS/NAM（免费但稍差）
  需要辐照度验证？→ SURFRAD/BSRN（最高质量地面数据）
  需要空间覆盖？→ NSRDB（全球卫星辐照度+Model Chain辅助变量）
  需要晴空模型输入？→ MERRA-2（REST2）或 SoDa（McClear/Ineichen-Perez）
  需要长期历史？→ ERA5（1940至今）或 MERRA-2（1980至今）
```

**训练问题**：

1. 为什么不能用卫星辐照度完全替代地面观测？
   → 虽然 Yang & Perez (2019) 证明两者在评估 NWP 预测质量方面近似等效，但 Murphy-Winkler 框架的严格检验显示它们在每个质量维度上都有差异。卫星辐照度是区域平均值（空间尺度不匹配），有系统偏差和欠色散——作为后处理和区域预测的参考可以，但作为验证的"真值"仍需谨慎。

2. ECMWF 比 NCEP 更准，为什么文献中用 NCEP 的论文反而更多？
   → 两个原因：①最早的 NWP 太阳预测研究在美国开展（Perez et al.），后续研究者倾向用相同数据方便对比。②NCEP 数据完全免费公开（美国纳税人资助），ECMWF 需要付费许可。这是 Ch2.9 可获取性原则的现实体现——不是最好的数据就是最被采用的数据。

3. 为什么教材说"不可复现的研究等于伪科学"？
   → 直接呼应 Ch2.4 的可证伪性原则（Popper）。如果一个预测方法使用私有数据、不公开代码，别人无法验证其结论是否成立——它在本质上不可证伪。不可证伪 = 不是科学。这听起来极端，但这正是教材作者一贯的立场：用公开数据+公开代码发表，敢让别人挑错。

4. 传统的"方法-尺度关联图"（Inman 2013）为什么过时了？
   → 两个原因：①技术进步——最新卫星分辨率达 0.5km/5min，高分辨率 NWP 达 200m/2min，两者都侵入了原本只属于天空相机的领地。②数据融合——统计降尺度和动力降尺度可以把粗分辨率 NWP 转化为细分辨率预测。关键认识：不要因为"NWP 是日前工具"就不尝试用它做小时内预测。

## 🔗 Ch7 逻辑链：三大基础预测方法的物理根基

**推演起点**：Ch6 告诉我们数据在哪，Ch7 告诉我们怎么用这些数据生成"原始预测"

```
基础预测方法 = 生成"未经后处理"的原始预测
  → 按重要性排序：NWP > 卫星 > 地面数据

为什么 NWP 排第一？
  → NWP 是唯一能"求解"大气运动方程的方法
    → 7 个原始方程：3个动量 + 温度 + 水汽 + 质量 + 状态
    → 但方程只描述 >10km 的过程
      → 更小尺度需要参数化
        → 辐射参数化看似最相关，实际不是瓶颈
          → 辐射传输物理已经很成熟
          → 真正瓶颈：云微物理参数化
            → 云在微米尺度形成，NWP网格是千米级
            → 微物理→对流→PBL→地表 全部互相耦合
              → 改一个影响全部 → 极难优化

卫星预测为什么只能做 <4h？
  → 核心假设：云场"冻结"（形态不变只做平移）
    → 光流法/块匹配 → 提取CMV → 平流辐照度场
    → 15-30分钟后云开始变形/消散/生成
    → 4小时后假设完全失效
      → 因为卫星方法不能"生成新云"
      → 只有 NWP 能模拟云的生命周期（凝结→碰并→降水→消散）

地面数据预测为什么最弱？
  → 只有时间维度信息（单站）或有限空间信息（传感器网络）
    → 空间相关性随距离快速衰减
    → >4h 后历史观测值对未来没有预测力
      → 只剩气候学（长期平均）有信息
    → 但时空方法（STAR/Kriging/GNN）在<1h内有用
      → 上游站点的"提前量"可以预见下游几分钟后的云

三种方法的关系：
  NWP → 日前框架（不可替代）
  卫星 → 日内补充（填补 NWP 更新间隔的空白）
  地面 → 超短期精修（分钟级实时校正）
  → 最优策略 = 三者融合（forecast combination）
```

**训练问题**：

1. 为什么辐射参数化方案不是NWP辐照度预测的瓶颈？
   → 因为辐射方案只是把云水含量转换为辐照度——物理已经很成熟。真正决定辐照度精度的是云水含量本身是否模拟准确，而这取决于云微物理、对流和PBL参数化方案。改进辐射方案的回报远小于改进云模拟。

2. 光流法的亮度恒定约束 I(x,y,t) = I(x+dx,y+dy,t+dt) 在什么情况下失效？
   → ①云形态变化（膨胀/收缩/合并/分裂）②新云生成或旧云消散 ③光照角度变化导致亮度变化（日出日落附近） ④云的三维运动被投影到二维（不同高度的云运动方向可能不同）。所有这些都会随时间加剧，导致CMV预测精度指数衰减。

3. 为什么单站ML模型（即使是LSTM）对>4h预测没有帮助？
   → LSTM能记住长时序的模式，但>4h后当前kappa和未来kappa的自相关gamma_h已经接近零——历史信息对未来没有预测力了。无论LSTM多复杂，信息论上不可能从"没有信息的输入"中提取信息。此时唯一的新信息来源是NWP（通过求解大气方程带来物理外推能力）。

## 🔗 Ch8 逻辑链：后处理的四象限——从原始预报到最终产品

**推演起点**：Ch7 生成了原始预报。但原始预报有系统偏差、缺乏不确定性量化。Ch8 解决这两个问题。

```
后处理的动机 = 两个"不匹配"
  ① 预报质量不匹配：原始预报有系统偏差
     → D2D 解决：回归(MOS)/Kalman滤波/降尺度
  ② 预报形式不匹配：预报员给概率，用户要确定性（或反过来）
     → P2D/D2P/P2P 解决

为什么 D2D 回归需要领域知识？
  → 把NWP变量直接丢进黑箱 vs 先用晴空指数变换
    → Ch4.5.1 证明了晴空归一化是最大武器
    → MOS的预测因子包含cos⁴Z…cosZ：这不是随意选择
      → 而是因为辐照度与天顶角的关系接近cos(Z)的多项式
    → 结论：后处理模型也必须尊重物理

为什么 Kalman 滤波比回归更适合运营？
  → 运营环境中天气模式不断变化
    → 批量回归的参数是"平均最优"→不适应当前天气
    → Kalman 滤波逐步更新→自动追踪偏差漂移
    → 但只能1步ahead → 多步需要h个独立滤波器

D2P 的核心假设：天气模式会重复
  → AnEn：当前NWP → 找历史相似NWP → 对应观测=集成成员
    → 本质是 nearest neighbors（机器学习的老朋友）
    → Pangu-Weather/GraphCast 也是"高级AnEn"
  → 穿衣法：把历史误差"穿"到当前预报上
    → 但误差非齐次（方差随天气变化）→需要条件采样

P2D 的反直觉结论：
  → 先做概率预报再提取确定性预报 > 直接做确定性预报
    → 因为概率预报的训练目标更丰富（CRPS vs MSE）
    → 提取时选对泛函很重要：MSE→均值, MAE→中位数
    → 中国四大电网惩罚方案各不同→最优泛函也不同

P2P 的核心——多样性(Diversity)：
  → MSE_组合 = 平均MSE - 多样性项
    → 多样性项 ∝ E[(Xⱼ-Xₖ)²]
    → 成员越不同→组合越好
      → 不能只靠大量ML模型（高相关→低多样性）
      → 应混合物理+数据驱动（Ch7的NWP+卫星+地面）
```

**训练问题**：

1. 为什么 MOS 用偏差(bias)而不是 GHI 作为预测目标？
   → 因为GHI的天顶角依赖性太强，模型会花大量容量学习日出日落规律。用偏差作为目标，天顶角多项式项能更高效地捕捉与角度相关的系统误差。同理Ch4.5.1用晴空指数去除确定性变化。

2. 为什么等权平均在预测组合中这么难被打败？(Forecast Combination Puzzle)
   → 因为最优权重在训练集上估计但在验证集上评估。权重估计有方差（sampling uncertainty），当成员数少或时间短时，估计的"最优"权重可能比1/m更差。Armstrong(2001)："除非有强证据支持不等权，否则用等权。"

3. 为什么降尺度的终极追求应该是概率化的？
   → 因为没有未来小尺度辅助变量（高分辨率的云/气溶胶），单条降尺度序列的ramp必然与实际错位→误差大。但多条降尺度序列联合构成预测分布→可以评估校准性(calibration)和锐度(sharpness)。这与Ch3的论点一致：不确定性存在时，概率预报总优于确定性预报。

4. AnEn 和深度学习天气模型（Pangu-Weather/GraphCast）本质上有什么共同点？
   → 都基于"天气模式会重复"的假设。AnEn显式搜索历史模式，DL隐式在训练中学习模式。差异在于DL能处理更高维的模式空间，且"搜索"是通过前向传播而非O(nm)距离计算。但两者都无法预测训练集中从未出现过的极端事件。
