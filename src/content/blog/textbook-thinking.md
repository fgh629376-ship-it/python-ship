## 二、逐章逻辑链

### Ch1-2：从"为什么预测"到"怎么判断好坏"

**核心因果链：**

1. 光伏出力取决于天气 → 天然不可控（间歇性）
2. 电网需要供需平衡 → 必须提前知道光伏出力 → **需要预测**
3. 预测有好坏之分 → Murphy 三维度：**一致性**（诚实）/ **质量**（准确）/ **价值**（有用）
4. 质量好 ≠ 价值高 → 用户不会用 = 零价值
5. 怎么避免"看似正确实则荒谬"的研究？→ 批判性思维三工具：
   - **剃刀**：如无必要，勿增实体（简单优先）
   - **叉子**：对任何论断追问"证据是什么？"
   - **扫帚**：警惕把不利事实藏起来

**训练问题：**

**Q1：一篇论文只报 $R^2 = 0.95$ 不报 RMSE，可能在"扫"什么？**

> **思考过程：** 光伏出力有强日周期（白天有、夜间无），这个周期本身就能贡献很高的 $R^2$。$R^2$ 衡量的是"模型解释了多少方差"——当数据有明显趋势时（如每天早上从 0 升到中午峰值再降回 0），即使预测很粗糙，$R^2$ 也能轻松到 0.9 以上。而 RMSE 衡量的是"预测偏离观测多少 $\text{W/m}^2$"，直接反映预测能力。只报 $R^2$ 不报 RMSE，很可能是因为 RMSE 数值很难看——模型可能在日出日落附近或多云天气下误差巨大，但被 $R^2$ 的"虚假繁荣"掩盖了。
>
> **答案：** 可能在"扫"真实的预测误差。$R^2 = 0.95$ 在光伏预测中几乎没有区分度（持续性模型都能做到），必须同时报 RMSE/MAE 才能反映真正的预测质量。这是典型的"奥卡姆扫帚"——选择性展示有利指标。

**Q2：XGBoost 的 RMSE 比 20 层 DNN 仅低 $1 \text{W/m}^2$，该选哪个？为什么？**

> **思考过程：** $1 \text{W/m}^2$ 的差异在统计上可能不显著（需要 DM 检验），但即使显著，还要考虑奥卡姆剃刀原则。XGBoost 是相对简单的集成树模型：训练快、可解释（能看特征贡献）、调参容易、部署简单。20 层 DNN 训练时间长数十倍、不可解释、对超参数敏感、容易过拟合、部署需要 GPU。$1 \text{W/m}^2$ 的提升完全不值得这些代价。
>
> **答案：** 选 XGBoost。理由：① $1 \text{W/m}^2$ 差异可能不显著（需 DM 检验）；② 奥卡姆剃刀——用复杂 10 倍的模型换不到 1% 的提升不划算；③ Markovics & Mayer (2022) 的 68 模型大比较证实 XGBoost/LightGBM 在典型数据量下整体最优。

### Ch3：从"混沌"到"为什么必须做概率预测"

**核心因果链：**

1. 天气是确定性系统（拉普拉斯妖）→ 但对初始条件极度敏感（混沌）
2. 观测永远有误差 → 初始条件永远不完美 → **确定性预测永远有误差**
3. 误差不可消除 → 必须量化不确定性 → **概率预测**
4. 概率预测的完整形式 = 预测分布 $F(x)$
   - 可以从中提取：分位数 $q_\tau$、区间 $[q_{0.1}, q_{0.9}]$、均值/中位数
5. 集成预测（多成员）= 经验分布 = 概率预测的离散近似
6. 评价概率预测：**校准**（统计一致性）+ **锐度**（区间越窄越好）
   - 先保证校准 → 再追求锐度（Gneiting 范式）

**关键公式：**
- CDF：$F_X(x) = P(X \leq x)$
- 分位数：$F^{-1}(\tau) = \inf\{x : F(x) > \tau\}$
- 混合高斯：$f(x) = \sum w_k \phi(x|\mu_k, \sigma_k)$

**训练问题：**

**Q3：辐照度能用高斯分布建模吗？为什么？**

> **思考过程：** 高斯分布 $N(\mu, \sigma^2)$ 的定义域是 $(-\infty, +\infty)$。但辐照度有物理约束：① 下界 $\geq 0$（负辐照度不存在）；② 上界约 $1.5 \times$ 晴空值（云增强效应极限）；③ 夜间恒为 0（这是一个点质量，不是连续分布）。用高斯建模会出现负辐照度的预测——物理上荒谬。而且辐照度分布通常是双峰的（晴天峰 $\kappa \approx 1$ + 阴天峰 $\kappa \approx 0.3$），单个高斯无法捕获。
>
> **答案：** 不能。三个原因：① 高斯延伸到负无穷，会产生负辐照度预测；② 夜间是点质量 $P(GHI=0)>0$，高斯是纯连续分布无法表示；③ 辐照度通常双峰，单高斯拟合很差。更好的选择：截断正态、混合高斯（2-3 组件）、或 Beta 分布（但 $\kappa>1$ 时 Beta 也不行）。

**Q4："80% 区间覆盖率 = 90%"是过分散还是欠分散？**

> **思考过程：** "80% 预测区间"意味着我们期望 80% 的观测落在区间内。如果实际覆盖率是 90%（比标称高），说明区间太宽了——比它声称的包含了更多观测。区间过宽 = 不确定性被高估 = 过分散（over-dispersed）。反过来，如果覆盖率 < 80% 则是欠分散（过度自信）。
>
> **答案：** 过分散（over-dispersed）。实际覆盖率 90% > 标称 80%，说明预测区间过宽，模型高估了不确定性。虽然"宁可过宽不要过窄"（过窄会漏掉极端事件），但过度分散意味着锐度不足——预测信息量低，对运营决策帮助有限。

### Ch4：光伏预测 vs 其他能源预测——独特性在哪？

**核心因果链：**

1. 光伏 vs 负荷/风力/电价：成熟度最低（2010s 才起步）
2. 光伏预测的**独特优势**：有晴空期望（其他领域没有）
   - 晴空模型 REST2：$$B_{nc} = E_{0n} \times T_R \times T_g \times T_o \times T_n \times T_w \times T_a$$
   - 晴空指数：$\kappa = \text{GHI} / \text{GHI}_{\text{clear}}$ → 去除天文周期 → 只剩天气信号
3. **GEFCom2014 关键事实**：唯一用晴空模型的团队 = 冠军
4. Model Chain（辐照度→功率）是光伏独有的转换链：
   - $k_t = G_h/E_0$ → 分离 → $G_c = B_n\cos\theta + R_dD_h + \rho_gR_rG_h$ → 温度 → 功率
5. 风功率：$P = \frac{1}{2}C_p\rho\pi R^2 W^3$（$P \propto W^3$ → 风速小误差被立方放大）
6. 电价：可以出现负值、极端尖峰 → 需要 VST 变换

**训练问题：**

**Q5：为什么晴空指数 $\kappa$ 比原始 GHI 更适合做预测模型的输入？**

> **思考过程：** 原始 GHI 包含两个成分：① 确定性的天文周期（太阳位置随时间变化，这是完全可计算的）；② 随机的天气影响（云、气溶胶等）。如果直接用 GHI，模型需要先"学会"太阳升起和落下这个已知事实——浪费模型容量在已知知识上。$\kappa = \text{GHI}/\text{GHI}_{\text{clear}}$ 用晴空模型除去了天文周期，剩下的纯粹是天气信号，而且归一化到 $[0, \sim1.5]$ 范围，消除了季节性。
>
> **答案：** 因为 $\kappa$ 去除了确定性的天文周期和季节变化（晴空模型已完全吸收），模型只需学习天气驱动的随机波动。用同样复杂度的模型学更难的东西 → 效率更高。这正是 GEFCom2014 冠军的核心策略——唯一用晴空模型的团队。

**Q6：上海和洛杉矶的 GHI 相关性很高，这意味着什么？是因果关系吗？**

> **思考过程：** 上海和洛杉矶都在北半球中纬度地区，太阳位置的季节变化模式几乎相同（夏天高冬天低）。原始 GHI 中天文周期占主导 → 两地 GHI 的高相关性完全来自共同的季节模式，不是因为一地的天气影响了另一地。如果用 $\kappa$（去季节性后），两地的相关性会大幅下降甚至接近零——因为云和天气模式完全不同。
>
> **答案：** 不是因果关系，是**虚假相关**（spurious correlation）。高相关性完全来自共同的天文周期（都在北半球 → 同季节同日照模式），不存在因果机制。这正是为什么必须用 $\kappa$ 而非 GHI 做分析——直接用 GHI 计算相关性会被季节性虚假相关误导。

### Ch5：从理论到实操——数据处理每一步都有讲究

**核心因果链：**

1. 辐照度分量的闭合关系：$G_h = D_h + B_n\cos Z$
2. k 指数体系（归一化工具）：
   - $\kappa = G_h/G_{hc}$（晴空指数，最重要）
   - $k_t = G_h/E_0$（浑浊度指数，更粗糙）
   - $k = D_h/G_h$（散射比）
3. 数据质量控制（QC）三步：
   - 自动测试：PPL/ERL 物理极限 → $-4 \leq G_h \leq 1.5E_{0n}\cos^{1.2}Z + 100$
   - 目视检查：散点图 + 热力图 + 晴空指数时序
   - 缺失值填充：卫星辐照度 > 统计插值
4. 晴空模型选择：REST2（最准）> McClear（中等）> Ineichen-Perez（最简）
5. MSE 缩放理论：$$\text{MSE}(r,x) = E[c^2] \cdot (1-\gamma_h^2) \cdot V(\kappa)$$
   - 三因子：可预测性 × 变异性 × 量级
6. 时间对齐陷阱：用错聚合方案 → RMSE 虚增 28%！
7. 运营四元组 $(S, R_f, L, U)$ 取代模糊的"预测范围"

**训练问题：**

**Q7：为什么 QC 必须在时间聚合之前进行？**

> **思考过程：** 假设 1 分钟数据中有一个异常尖峰（如传感器故障导致 GHI = $5000 \text{W/m}^2$）。如果先聚合到小时平均，这个尖峰被 60 个正常值稀释——小时均值可能只高出几十 $\text{W/m}^2$，落在 QC 阈值内，逃过检测。但如果先在 1 分钟分辨率上做 QC，尖峰远超 PPL 上限（$1.5E_{0n}\cos^{1.2}Z + 100$），立刻被标记剔除。聚合后再检验 = 信息损失 → 漏检。
>
> **答案：** 因为时间聚合（平均）会**稀释异常值**，使其隐藏在正常范围内，逃过 QC 检测。1 分钟的极端尖峰在小时平均中被 59 个正常值冲淡。所以必须先 QC 再聚合——在最高分辨率上捕获异常，然后用干净数据做时间平均。

**Q8：REST2 和 Ineichen-Perez 性能差异大吗？选择依据是什么？**

> **思考过程：** Ch5.4.5 用 SURFRAD 7 站 4 年数据比较了三个模型。最重要的发现是：**不同晴空模型之间的差异远小于不同站点之间的差异**。也就是说，站点的气候特征（沙漠 vs 海洋性）对预测难度的影响远大于晴空模型选择。REST2 在 75 个模型中排名最高，但 Ineichen-Perez 排中等也够用。
>
> **答案：** 差异不大——站点间差异 >> 模型间差异。选择依据：① 追求最高精度 + 有再分析数据 → REST2（9 个输入参数，需 MERRA-2）；② 最省事 + 不限时间范围 → Ineichen-Perez（只需 Linke 浑浊度因子，几行代码）；③ 需要在线产品 → McClear（SoDa 网站免费，但不开源）。实际差异远小于预期——奥卡姆剃刀支持简单方案。

### Ch6：数据是预测的天花板

**核心因果链：**

1. 三种数据源各有所长：
   - **地面观测**：最准（5% 不确定度），但密度极低（中国仅约 100 站）
   - **卫星辐照度**：覆盖广（全球中低纬），精度中等，5min/0.5km
   - **NWP/再分析**：时空最全 + 变量自洽，但精度最低
2. 代表性数据集三要素：**覆盖**（时空气候天况）/ **一致性**（尺度匹配）/ **可获取**（可复现）
3. **不可复现 = 伪科学** → 必须用公开数据集
4. 关键数据库：BSRN（全球 70+ 站）、SURFRAD（美国 7 站，最高质量）、NSRDB（全球，含晴空值）

**训练问题：**

**Q9：NWP 网格点代表 5-50 km² 平均，地面观测是单点。直接比较会有什么问题？**

> **思考过程：** 云的空间变率很高——一朵积云可能只有 500m 宽。NWP 网格点代表的是 10km$\times 10$km 区域的平均辐照度，而地面站测的是**单点**。如果恰好有一朵云遮住了地面站但没遮住旁边区域，地面观测 GHI 很低但 NWP 网格平均 GHI 还不错——这个"误差"不是 NWP 预测不好，而是空间尺度不匹配造成的代表性误差。
>
> **答案：** 会引入**代表性误差**（representativeness error）。地面单点观测包含 NWP 无法分辨的亚网格尺度变率（特别是云导致的快速波动）。这种误差不是预测方法的问题，而是验证方法的缺陷。解决方案：① 用多站平均来匹配 NWP 网格尺度；② 用卫星辐照度（本身就是面积平均）做验证；③ 在评估中考虑代表性误差的贡献。

**Q10：为什么 ERA5 再分析不能直接替代地面观测做预测验证？**

> **思考过程：** 再分析是用"冻结"模型重新分析历史数据，其辐照度输出也是模型产物——有自己的系统偏差和误差特征。用模型产物验证另一个模型，等于用"近似真值"代替"真值"。ERA5 的辐照度精度（~31km 分辨率）明显低于高质量地面观测（5% 不确定度）。用 ERA5 验证可能：① 低估某些误差（如果两个模型有相同的偏差源）；② 引入虚假结论。
>
> **答案：** 因为 ERA5 辐照度本身是模型产物，不是真实观测。它有自己的系统偏差（特别是云的模拟不够精确），分辨率也较粗（31km）。用模型验证模型 → 可能系统性低估误差。地面观测虽然覆盖有限，但它是最接近"真值"的参考。折中方案：高质量卫星辐照度（如 NSRDB）可以作为"准真值"用于区域验证。

### Ch7：三大基础预测方法的物理根基

**核心因果链：**

1. **NWP（日前唯一选择）**：
   - 7 个原始方程：$\frac{D\theta}{Dt} = \frac{\theta}{T}\frac{Q}{c_p}$，$p = \rho RT$
   - 6 种参数化：辐射/云微物理/对流/PBL/地表/气溶胶
   - 瓶颈不是辐射方案，而是**云微物理**
   - CFL 稳定性：$\Delta t \leq \Delta x / c$ → 网格越细，计算量越大
   - 数据同化（4D-Var）：融合观测到初始条件
2. **卫星（日内补充，< 4h）**：
   - 云运动矢量 CMV → 冻结假设 → 15-30 min 后失效
   - 三类反演：物理（FARMS）/ 半经验（Heliosat）/ ML
3. **地面统计（超短期，< 1h）**：
   - 持续性参考：$\kappa(t+h) = \kappa(t)$
   - CLIPER：$\rho_t = \gamma_h \kappa_{t-h} + (1-\gamma_h)E(\kappa)$（数学最优）
   - 时空方法（STAR/克里金/GNN）略优但仍受限

**训练问题：**

**Q11：为什么"纯数据驱动可以替代 NWP 做日前预测"是幻想？**

> **思考过程：** 日前预测 = 提前 24-36 小时。此时晴空指数 $\kappa$ 的时序自相关 $\gamma_h$ 已经接近零——也就是说，**当前 $\kappa$ 对 24h 后的 $\kappa$ 几乎没有预测力**。纯数据驱动方法（LSTM/Transformer）只能从历史数据中提取信息，但信息论告诉我们：不可能从"没有信息的输入"中提取信息。NWP 之所以能做日前预测，是因为它求解大气原始方程——从当前大气状态物理外推未来状态，引入了历史数据中不存在的新信息。
>
> **答案：** 因为 $\gamma_h \to 0$（24h 后自相关消失），历史数据中**不包含**对未来 24h 天气的信息。数据驱动只能挖掘数据中已有的信息，而 NWP 通过物理方程**创造**新信息（外推大气运动）。这是信息论的硬约束，不是模型不够复杂的问题——即使用 100 层 Transformer 也突破不了。

**Q12：LSTM 在 > 4h 预测中为什么失效？**

> **思考过程：** LSTM 的核心能力是记住长时序模式。但晴空指数 $\kappa$ 的自相关 $\gamma_h$ 随提前量 $h$ 快速衰减。在 $h < 1h$ 时 $\gamma_h$ 较高，LSTM 可以利用近期的 $\kappa$ 值做有效预测。但 $h > 4h$ 后 $\gamma_h \approx 0$——这意味着当前 $\kappa$ 和未来 $\kappa$ 统计独立，历史信息对预测毫无帮助。LSTM 的"记忆"变得毫无价值，因为记住的东西和未来无关。
>
> **答案：** 因为 $\kappa$ 的自相关 $\gamma_h$ 在 4h 后接近零。LSTM 再强的记忆能力也无法从统计独立的输入中提取预测信息。此时唯一的新信息来源是 NWP（通过物理方程而非统计相关性）。这也解释了为什么 Ch4 的"方法-尺度"图中，统计方法在 > 4h 后被 NWP 全面超越。

### Ch8：后处理——从原始预报到最终产品

**核心因果链：**

后处理 = 不改变预测量（GHI 还是 GHI），只提升质量。按输入/输出分四象限：

**D2D（确定性→确定性）：**
- 回归：$\hat{y} = \beta_0 + \beta_1 x$（融入领域知识 > 黑箱 ML）
- Kalman 滤波：$Y_t = \mathbf{x}_t^T\beta_t + \varepsilon_t$（在线更新，运营追踪）
- 降尺度：追求概率化（单条降尺度序列的 ramp 必然错位）

**P2D（概率→确定性）：**
- 选对统计泛函：MSE → 均值，MAE → 中位数，Pinball → $\tau$ 分位数
- 组合预测：$\text{MSE}_{\text{combo}} = \frac{1}{m}\sum\text{MSE}_j - \text{多样性项}$
- **多样性是关键**：混合物理 + 数据驱动 > 大量同质 ML 模型

**D2P（确定性→概率）：**
- 模拟集成 AnEn：天气模式会重复 → 找历史相似情况
- 穿衣法：$N(0, \sigma_t^2)$（历史误差 → 不确定性估计）
- 分位数回归 / GAMLSS：$D(\mu, \sigma, \nu, \tau)$

**P2P（概率→概率）：**
- EMOS：均值 $= w_0 + \sum w_j x_j$，方差 $= \beta_0 + \beta_1 S_t^2$
- BMA：$g_t(x) = \sum w_j f_{tj}(x)$
- 核心原则：**先概率再确定性 > 直接确定性**

**训练问题：**

**Q13：等权平均为什么这么难打败？**

> **思考过程：** 组合预测的 MSE 分解为：$\text{MSE}_{\text{combo}} = \frac{1}{m}\sum\text{MSE}_j - \text{多样性项}$。多样性项 $= E[(X_j - X_k)^2] \geq 0$，是非负的——这意味着组合的 MSE **一定不大于**成员平均 MSE。等权平均自动获得这个保证，不需要任何训练数据来估计权重。而不等权需要估计 $w_j$，估计本身有噪声——特别是样本小时，不等权可能因为权重估计不稳定反而更差。Armstrong 法则总结得好："除非有强证据支持不等权，否则用等权。"
>
> **答案：** 因为 MSE 分解的多样性项 $\geq 0$（数学保证），等权组合的误差一定不超过成员平均误差。不等权需要估计权重，估计噪声可能抵消理论优势。等权 = 零估计风险 + 多样性红利。这就是为什么 68 个 ML 模型的等权平均很难被任何单模型或不等权方案打败。

**Q14：Kalman 滤波只能做 1 步 ahead，多步怎么办？**

> **思考过程：** Kalman 滤波的递推结构是：观测到来 → 更新状态 → 预测下一步。它天然只能预测 $t+1$。如果要预测 $t+2, t+3, \ldots, t+h$，不能简单地把 1 步预测的输出当成输入再递推（误差会快速累积）。Yang et al. (2017c) 发现的解决方案是：为每个提前量 $h$ 训练一个独立的 Kalman 滤波器。$h=1$ 一个滤波器，$h=2$ 一个，$\ldots$ 各自独立追踪不同提前量的偏差特征。
>
> **答案：** 为每个提前量 $h$ 建立独立的 Kalman 滤波器（$h$ 个并行滤波器）。每个滤波器只负责自己的提前量，各自追踪偏差。这避免了多步递推的误差累积问题。Yang et al. (2017c) 首次在光伏预测中揭示了这个问题和解决方案。

### Ch9：确定性验证——"赢了指标"≠"赢了预测"

**核心因果链：**

1. 单一指标评优**毫无意义**：
   - Novice 赢 MBE，Optimist 赢 MAE，Statistician 赢 RMSE
   - 同一组数据，三个指标选出三个不同的"最佳"
2. **为什么？** → 不同指标惩罚不同类型的误差
   - RMSE 惩罚大误差（平方），MAE 对所有误差等权，MBE 正负相消
3. Skill Score：$S^* = 1 - S_{\text{fcst}} / S_{\text{ref}}$
   - 参考方法用 CLIPER（$\gamma_h \kappa_{t-h} + (1-\gamma_h)E(\kappa)$），不用 persistence
   - CLIPER 严格优于 persistence 和 climatology（数学证明）
   - **打不过 CLIPER 的模型没有存在价值**
4. Murphy-Winkler 分布导向：看 f-o 联合分布的三个切面
5. **质量 vs 价值**：9% 的质量差异 → 5 倍惩罚差异（非线性映射）

**训练问题：**

**Q15：模型 A 的 MAE = 50，模型 B 的 MAE = 52，能说 A 更好吗？**

> **思考过程：** $2 \text{W/m}^2$ 的差异可能来自随机波动。在一个测试集上 A 恰好比 B 好 2，换一个测试集可能 B 反超。统计学要求我们区分"真实差异"和"随机噪声"。Diebold-Mariano (DM) 检验正是做这件事：计算 $d_t = L(e_{A,t}) - L(e_{B,t})$（两模型在每个时刻的损失差），检验 $H_0: E[d_t] = 0$。DM 统计量近似 $N(0,1)$，p-value < 0.05 才能说差异显著。
>
> **答案：** 不能直接说 A 更好。必须做 **Diebold-Mariano 检验**：$H_0: E[d_t] = 0$，$d_t = |e_{A,t}| - |e_{B,t}|$（MAE 对应绝对误差损失）。只有 p-value 足够小（如 < 0.05），才能在统计意义上说 A 显著优于 B。光伏预测论文极少做此检验——大多数只比较数字，这在统计上不严谨。

**Q16：为什么"MAPE 最优的预测反而惩罚更高"？**

> **思考过程：** MAPE 惩罚的是百分比误差——在辐照度接近零时（日出日落附近），即使绝对误差很小（如 $5 \text{W/m}^2$），但因为分母很小（如 $10 \text{W/m}^2$），MAPE 爆到 50%。MAPE 最优的模型会"优化"这些边缘时段，把低辐照度时段的预测调得很准——但这些时段发电量极低，对电网几乎没价值。同时，它可能忽略了中午高辐照度时段的大绝对误差（$500 \text{W/m}^2$ 误差 $50 \text{W/m}^2$，MAPE 仅 10%），而这些时段的误差才是不平衡罚款的主要来源。
>
> **答案：** 因为 MAPE 对低辐照度时段（日出日落）的小误差赋予过高权重，导致模型"优化"了经济价值最低的时段，却忽略了高辐照度时段的大绝对误差——而电网罚款通常与绝对偏差成正比。这就是 Ch9.5 的核心发现：**质量（指标好）和价值（经济收益）是非线性关系**，9% 的质量差异可以导致 5 倍的惩罚差异。

### Ch10：概率验证——"正确的不确定性"比"准确的点值"更重要

**核心因果链：**

1. 电网需要备用容量 → 备用量取决于不确定性大小 → **概率预测的核心价值**
2. Gneiting 范式：**最大化锐度，同时保持校准**
   - 校准 = PIT 服从均匀分布：$F_t(y_t) \sim U(0,1)$
   - 锐度 = 预测区间越窄越好
   - 陷阱：气候态完美校准但零锐度 → 无价值
3. 四大严格适当评分规则：
   - **CRPS**：$$\text{crps}(F,y) = \int_{-\infty}^{\infty}[F(x) - \mathbb{1}(x \geq y)]^2\,dx$$
   - CRPS = MAE 的概率推广 = Brier Score 对所有阈值的积分
   - **IGN**：$-\log f(y)$（局部评分，只看 PDF 在观测点的值）
4. **一致性原则**：用什么评 → 就用什么训练
   - CRPS 评估 → CRPS 训练 ✅
   - IGN 评估 → 似然训练 ✅
   - 混用 → 次优 ❌

**训练问题：**

**Q17：CRPS 的单位是什么？和 MAE 一样吗？**

> **思考过程：** CRPS 的定义是 $\int[F(x) - \mathbb{1}(x \geq y)]^2 dx$。被积函数 $[F(x) - \mathbb{1}]^2$ 是无量纲的（概率差的平方），而积分变量 $dx$ 的单位与预测变量相同（$\text{W/m}^2$）。所以 CRPS 的单位 = 预测变量的单位。当预测分布退化为点预测 $\delta(\hat{y})$ 时，$\text{CRPS} = |y - \hat{y}| = \text{MAE}$。两者单位一致，数值上 CRPS 也可以直接与 MAE 比较。
>
> **答案：** CRPS 的单位与预测变量相同（$\text{W/m}^2$），和 MAE 一样。这是 CRPS 的重要优势——它既是概率评分又保持了物理可解释性。当概率预测退化为点预测时，CRPS 精确等于 MAE，所以两者可以直接比较数值大小。

**Q18："严格适当"意味着什么？如果评分规则不严格适当会怎样？**

> **思考过程：** "适当"(proper) 意味着：预测者按自己的**真实信念**出预测时，期望得分是最优的——没有动机歪曲预测。"严格"(strictly) 更进一步：**只有**按真实信念出预测才能达到最优，任何偏离都会变差。如果评分规则不严格适当，预测者可以通过"策略性作弊"（hedging）获得更好的分数——比如明知会下雨但报 60% 而非 80% 以降低期望损失。这会污染验证结果：得分高不代表预测好，可能只是"玩"得好。
>
> **答案：** "严格适当"意味着**只有按真实信念出预测才能最优化期望分数**，任何策略性偏离都会变差 → 不鼓励作弊。如果评分规则不严格适当，预测者有动机发布不代表真实信念的预测（hedging），验证结果被污染——高分 ≠ 好预测。这就是为什么 CRPS、IGN、Brier Score 被推荐：它们都是严格适当的。

### Ch11：辐照度→功率——Model Chain 的误差传播

**核心因果链：**

1. 电网需要**功率**预测，不是辐照度预测
2. 三类转换方法：
   - **直接回归**：NWP 变量 → ML → 功率（需要历史数据）
   - **物理 Model Chain**：分离 → 转置 → 反射 → 温度 → PV 模型 → 损耗
   - **混合法**：物理前半段 + ML 后半段（通常最优）
3. 特征重要性排序：**晴空信息 > 特征工程 > 概率/集成 > 模型选择**
4. 分离模型演进：ERBS(1982) → BRL(2010) → ENGERER2(2015) → YANG4(2021,当前最优)
5. **端到端优化必要性**：
   - 分离模型 A 高估散射 5% → 转置模型 B 对散射敏感 → 整体误差放大
   - 最佳个体模型组合 ≠ 最佳 Model Chain
6. 温度影响：$T_{\text{cell}} = T_{\text{amb}} + \frac{\text{NOCT}-20}{800} \times G_c$
   - $\gamma \approx -0.4\%/°C$ → 40°C 沙漠比 25°C STC 功率损失约 6%

**训练问题：**

**Q19：新电站没有历史数据，只能用哪种方法？为什么？**

> **思考过程：** 三类方法的数据需求不同。直接回归需要历史的"输入-功率"训练数据——新站没有。物理 Model Chain 只需要电站的设计参数（面板型号、倾角、方位角、逆变器规格）加上辐照度预测——不依赖历史功率数据。混合法的 ML 后半段也需要训练数据。所以只有纯物理 Model Chain 可以在零历史数据条件下工作。
>
> **答案：** 只能用**物理 Model Chain**。因为它只需要电站设计参数（面板参数、安装角度等）和辐照度预测输入，不需要历史功率数据进行训练。这正是两阶段预测的"数据可扩展性"优势——新电站无需等待数据积累就能立即出预测。等积累了足够历史数据后，可以切换到混合法进一步优化。

**Q20：为什么"晴空信息 > 模型选择"？这和 Ch4 的哪个结论呼应？**

> **思考过程：** Ch11.1 的特征重要性排序表明：给模型提供晴空指数 $\kappa$（而非原始 GHI）带来的提升，比换一个更复杂的模型带来的提升大得多。原因在于：$\kappa$ 去除了已知的天文信号，让模型专注于天气信号——这是信息层面的提升。而换模型只是在拟合能力上微调——同样的信息输入，不同模型能提取的信息量差异有限。
>
> **答案：** 因为晴空归一化是**信息层面**的提升（去除已知信号 → 增加有效信息密度），而模型选择只是**拟合能力**的微调。这直接呼应 Ch4.6.2："给 LSTM 晴空指数 >> 让 LSTM 自己学太阳升落"，也呼应 GEFCom2014 的结论：唯一用晴空模型的团队 = 冠军。**领域知识的价值 > 模型复杂度的价值**。

### Ch12：层次预测与 Firm Power——教材的终极闭环

**核心因果链：**

1. 电力系统天然分层：互联 → 输电区 → 电站 → 逆变器
2. 各层独立预测 → 求和不一致（incoherent）→ 调度混乱
3. 层次调和：$$\tilde{\mathbf{y}} = \mathbf{S} \cdot \mathbf{P} \cdot \hat{\mathbf{y}}$$
   - 5 种方法：BU / OLS / WLS / HLS / **MinT-shrink**（最优）
   - 理论保证：调和后每层预测 ≥ 独立预测
4. **Firm Power（终极目标）** = 预测 + 储能 + 调度 → 可调度太阳能
   - Firm Forecasting（消除预测误差）→ 成本 $\sim 2\times$
   - Firm Generation（消除间歇性）→ 成本 $\sim 3\times$
5. 四大 enablers：
   - **储能**：核心但贵 → 需要优化容量
   - **地理平滑**：分散选址 → 自然界的"组合预测"
   - **负荷塑形**：让需求适应供给（需求侧革命）
   - **超配 + 弃电**：反直觉但必要（所有实证研究一致）
6. **闭环**：预测越准 → 储能需求越少 → firm 成本越低 → **预测研究有直接经济价值**

**训练问题：**

**Q21：为什么"超配 + 弃电"是最低成本 firm power 的必要组成？**

> **思考过程：** 考虑边际成本。储能的目的是填补光伏出力不足的时段。为了覆盖最后 5% 的极端情况（连续阴天），需要大量额外储能——但这些储能一年可能只用几次，边际利用率极低。而如果光伏容量超配 1.5 倍，晴天中午的多余发电可以通过弃电处理（不存储），但大部分时间多余的容量减少了储能需求。弃掉 10% 的电量换来储能需求减少 30%——弃电的边际成本远低于等量储能的边际成本。
>
> **答案：** 因为**弃电的边际成本 < 等量储能的边际成本**。超配使得大部分时间光伏出力有余量，减少了对储能的依赖。虽然弃掉了部分发电（看似浪费），但省下的储能投资远超弃电损失。所有实证研究（包括 CAISO 案例）一致证明：最优 firm power 方案的超配比 $\sim 1.13\text{-}1.54\times$，对应的弃电率是系统总成本最低的必要代价。

**Q22：从 Ch1 到 Ch12 写出完整因果链（5 步以内）。**

> **思考过程：** 提炼全书 12 章为 5 个核心步骤，每步都是前一步的逻辑必然：
>
> **答案：**
> 1. **光伏间歇性 → 必须预测**（Ch1-2：问题定义）
> 2. **不确定性不可消除 → 概率预测 + 晴空归一化**（Ch3-5：理论与数据基础）
> 3. **NWP + 后处理 → 生成并优化预测**（Ch6-8：方法论）
> 4. **严格验证 → 确保预测可信**（Ch9-10：质量保证）
> 5. **Model Chain + 层次调和 + 储能 → 可调度太阳能**（Ch11-12：终极应用）
>
> 一句话：**间歇性 → 概率预测 → NWP+后处理 → 严格验证 → Firm Power**。
