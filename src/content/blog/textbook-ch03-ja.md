## 3.1 物理世界の決定論的な本質

**核心となる推論の連鎖**：
1. すべてのカオス系は決定論的だ（前提1）
2. 天気はカオス系だ（前提2）
3. ゆえに天気は決定論的だ（結論）

これがラプラスの悪魔（Laplace's Demon）の思考実験だ——系の正確な初期状態とすべての物理法則を知っていれば、未来は完全に決定される。

しかし**決定論 ≠ 予測可能性**。カオス系は初期条件に対して極めて敏感であり、誤差は指数関数的に成長する。これが天気予報の理論的な限界の源泉だ。

予測誤差の二つの源：
1. **不完全な初期条件**（観測には常に誤差が伴う）
2. **不完全な力学法則**（パラメータ化スキームは近似に過ぎない）

## 3.2 不確実性と分布予測

### 3.2.1 確率分布の基礎

- CDF：$F_X(x) = P(X \leq x)$、値域 $[0,1]$
- PDF：$f_X(x) = F'_X(x)$、**値は1より大きくなり得る**（よくある誤解）
- 日射量には物理的な上限・下限があるため、$\pm\infty$ まで広がるガウス分布は不適切

### 3.2.2 予測分布 (Predictive Distribution)

確率的予測の中心的な形式。自然は真の分布 $G_i$ を選択し、予測者は $F_i$ を発行する。

分位数の定義：
- $\tau$ 分位数 $= F^{-1}(\tau) = \inf\{x : F(x) > \tau\}$
- $\tau$ は確率水準（0〜1）であり、分位数の値は具体的な日射量の数値
- ⚠️ $X$ の範囲も $[0,1]$ の場合（例：クリアスカイ指数 $\kappa$）、$\tau$ と分位数の値は非常に混同しやすい

### 3.2.3 理想的な予測分布（Gneiting et al., 2007）

確率的予測の検証における礎となる論文。中核的なパラダイム：

> **較正（calibration）の制約のもとで、鋭度（sharpness）を最大化する**

- **較正**：予測分布と観測の統計的な一致性（「80%区間」と言えば、実際に80%をカバーしていること）
- **鋭度**：予測分布の集中度（較正が達成されていることを前提に、区間は狭いほど良い）

よくある誤り：
- 鋭度だけを追い求める → 区間が狭すぎ、実際のカバー率が名目上の水準を大幅に下回る
- 較正だけを追い求める → 気候学的予測でも較正は達成できるが、スキルが全くない

### 3.2.4 予測分布の四つのタイプ

| タイプ | 例 | 特徴 | 太陽光発電への応用 |
|--------|-----|------|-----------------|
| パラメトリック | ガウス $N(\mu,\sigma)$ | 固定パラメータで記述 | シンプルだが打ち切りが必要（日射量≥0） |
| セミパラメトリック | 混合ガウス（2〜3成分） | 多峰性を捉える | 日射量は自然に二峰性（晴天/曇天） |
| ノンパラメトリック | KDEカーネル密度推定 | 形状を仮定しない | 柔軟だが帯域幅の選択が必要 |
| 経験的 | ECDF | 完全に無パラメータ | 最もシンプルだが大量サンプルが必要 |

### 3.2.5 分布予測のその他の表現形式

- **分位数予測**：特定の $\tau$ 値に対応する分位数を報告する（例：$q_{0.1}, q_{0.5}, q_{0.9}$）
- **区間予測**：$(1-\alpha)\times100\%$ 中央区間 = $[\alpha/2, 1-\alpha/2]$ 分位数
- **アンサンブル予測**：複数の確定的予測の集合

## 3.3 アンサンブル予測

### 3.3.1 気象アンサンブルの種類

不確実性の源に応じて三種類：

| タイプ | 原理 | 例 | 対応する不確実性 |
|--------|------|-----|----------------|
| 動力学的アンサンブル | 初期条件の摂動 | ECMWF EPS（50+1メンバー） | データ不確実性 |
| 貧者のアンサンブル | 異なるモデルを使用 | ECMWF + GFS + NAM | プロセス不確実性 |
| 確率的パラメータ化 | 異なる物理スキーム | 異なる雲微物理スキーム | パラメータ不確実性 |

### 3.3.2 統計的アンサンブル予測

アンサンブルを生成する三つの統計的戦略：

**データアンサンブル**：
- 時系列長の摂動（異なる学習期間長でトレーニング）
- ブートストラップ集約（baggingの統計的基盤）
- 複数のデータソース（地上・衛星・再解析データによる同一変数の異なるバージョン）
- 空間アンサンブル（近隣のピクセル/地点の予測をアンサンブルに加える）

**プロセスアンサンブル**：
- 豊富な時系列モデルの選択肢：ARIMAファミリー、ETSファミリー
- 各種分解手法：STL / X11 / SEATS / フーリエ / ウェーブレット
- 分解 → 予測 → 再構成が古典的なワークフロー
- TBATS = 三角（フーリエ） + Box-Cox + ARMA誤差 + トレンド + 季節性

**パラメータアンサンブル**：
- ARIMAの $(p,d,q)$ やETSの状態パラメータを調整
- 最適パラメータ化と矛盾する？——しかし最適でないパラメータ化は不確実性定量化に価値がある
- 類推：NWPのアンサンブルメンバーも最良の推測ではないが、不確実性の定量化に有用だ

### 3.3.3 アンサンブル学習

統計的アンサンブルとアンサンブル学習の違い：本質的には同じだが、用語の出所が異なるフィールドだ。異なるコミュニティが独立して同じ概念を発明したのが核心だ。

#### 3.3.3.1 Bagging（ブートストラップ集約）

- 複数のブートストラップサンプルを作成 → それぞれで基本学習器を訓練 → 予測を平均化
- **分散を低減**（決定木など高分散モデルに適している）
- 基本学習器は**並列訓練が可能**（embarrassingly parallel）
- 保証：アンサンブルのMSE ≤ 基本学習器の平均MSE。ただし**最良の基本学習器より優れているとは限らない**
- 多様性が鍵——基本学習器の差異が大きいほど、アンサンブルの効果は高い
- Out-of-bagサンプルを内部評価に活用できる

#### 3.3.3.2 Boosting

- 逐次的に構築し、各ラウンドでは前のラウンドで誤分類されたサンプルに焦点を当てる
- AdaBoost.M1のコアステップ：
  1. 等ウェイトで初期化：$w_i = 1/n$
  2. 弱分類器を訓練 → 誤分類率 $\eta_j$ を計算
  3. $\alpha_j = \log[(1-\eta_j)/\eta_j]$ を計算
  4. 誤分類サンプルのウェイトを $\exp(\alpha_j)$ 倍する
- **バイアスを低減**（浅い木など高バイアスモデルに適している）
- 回帰の場合：指数損失を二乗/絶対値/Huber損失に置き換える
- XGBoostはboostingの現代的な実装

#### 3.3.3.3 Stacking

- 二層構造：基本学習器（Level 0） + スーパー学習器（Level 1）
- スーパー学習器は**交差検証**によって訓練される（追加のホールドアウトセット不要）
- 正則化回帰がスーパー学習器としてよく使われる
- 単純な後処理との違い：stackingは二段階のデータ分割を必要としない

## 3.4 深考 (After Thought)

- 統計的アンサンブルとアンサンブル学習ツールボックスの核心的な違い：**成分予測**にアクセスできるかどうか
  - 統計的アンサンブル：外部で組み合わせるため、各成分を確認できる
  - アンサンブル学習ツールボックス：通常は最終的な組み合わせ結果のみを出力
- 確率予測から確定的予測への変換は**単純ではない**：
  - 平均を使うか中央値を使うか？ → 評価指標の選択に影響する
  - 確定的予測に平均を使うなら、MSEで評価すべき
  - 確定的予測に中央値を使うなら、MAEで評価すべき
  - ほとんどの太陽光発電予測者はこれらの問いを立てたことがない
