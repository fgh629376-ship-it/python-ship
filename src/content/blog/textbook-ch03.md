## 3.1 物理世界的确定性本质

**核心推理链**：
1. 所有混沌系统都是确定性的（前提一）
2. 天气是混沌系统（前提二）
3. 因此天气是确定性的（结论）

这就是拉普拉斯妖（Laplace's Demon）的思想实验——如果知道系统的精确初始状态和所有物理定律，未来就完全确定。

但**确定性 ≠ 可预测**。混沌系统对初始条件极度敏感，误差会指数增长。这就是天气预测的理论极限来源。

预测误差的两个来源：
1. **初始条件不完美**（观测永远有误差）
2. **动力学定律不完美**（参数化方案是近似）

## 3.2 不确定性与分布预测

### 3.2.1 概率分布基础

- CDF：F_X(x) = P(X ≤ x)，值域 [0,1]
- PDF：f_X(x) = F'_X(x)，**值可以 > 1**（这是常见误解）
- 辐照度有物理上下界，Gaussian 分布（延伸到 ±∞）不适合

### 3.2.2 预测分布 (Predictive Distribution)

概率预测的核心形式。Nature 选择真实分布 Gᵢ，预测者发布 Fᵢ。

分位数定义：
- τ 分位数 = F⁻¹(τ) = inf{x : F(x) > τ}
- τ 是概率水平（0-1），分位数值是具体的辐照度值
- ⚠️ 当 X 范围也是 [0,1] 时（如晴空指数 κ），极易混淆 τ 和分位数值

### 3.2.3 理想预测分布（Gneiting et al., 2007）

概率预测验证的基石论文。核心范式：

> **在校准 (calibration) 约束下，最大化锐度 (sharpness)**

- **校准**：预测分布与观测的统计一致性（说 80% 区间，真的覆盖 80%）
- **锐度**：预测分布的集中程度（区间越窄越好，前提是校准达标）

常见错误：
- 只追求锐度 → 区间太窄，实际覆盖率远低于标称
- 只追求校准 → 气候学预测也能校准，但没有技巧

### 3.2.4 预测分布的四种类型

| 类型 | 示例 | 特点 | 光伏应用 |
|------|------|------|---------|
| 参数型 | 高斯 N(μ,σ) | 用固定参数描述 | 简单但需截断(辐照度≥0) |
| 半参数型 | 混合高斯(2-3个) | 捕获多峰 | 辐照度天然双峰（晴/阴） |
| 非参数型 | KDE 核密度估计 | 不假设形状 | 灵活但需选带宽 |
| 经验型 | ECDF | 完全无参数 | 最简单，需大量样本 |

### 3.2.5 分布预测的其他表示

- **分位数预测**：报告特定 τ 值对应的分位数（如 q₀.₁, q₀.₅, q₀.₉）
- **区间预测**：(1-α)×100% 中央区间 = [α/2, 1-α/2] 分位数
- **集成预测**：多个确定性预测的集合

## 3.3 集成预测

### 3.3.1 气象集成类型

按不确定性来源分三种：

| 类型 | 原理 | 示例 | 对应不确定性 |
|------|------|------|------------|
| 动力集成 | 扰动初始条件 | ECMWF EPS (50+1成员) | 数据不确定性 |
| 穷人集成 | 不同模型 | ECMWF+GFS+NAM | 过程不确定性 |
| 随机参数化 | 不同物理方案 | 不同云微物理方案 | 参数不确定性 |

### 3.3.2 统计集成预测

三种生成集成的统计策略：

**数据集成**：
- 时序长度扰动（用不同历史长度训练）
- Bootstrap 聚合（bagging 的统计基础）
- 多数据源（地面+卫星+再分析的同一变量不同版本）
- 空间集成（邻近像元/站点的预测加入集成）

**过程集成**：
- 大量时序模型可选：ARIMA族、ETS族
- 各种分解方法：STL / X11 / SEATS / 傅里叶 / 小波
- 分解→预测→重构是经典工作流
- TBATS = 三角(Fourier) + Box-Cox + ARMA误差 + 趋势 + 季节

**参数集成**：
- 调整 ARIMA 的 (p,d,q) 或 ETS 的状态
- 与最优参数化相矛盾？——但非最优参数化对不确定性量化有价值
- 类比：NWP 集成成员也不是最佳猜测，但对量化不确定性有用

### 3.3.3 集成学习

统计集成 vs 集成学习的区别：本质相同，但术语来自不同领域。核心是不同社区独立发明了相同概念。

#### 3.3.3.1 Bagging (Bootstrap Aggregating)

- 创建多个 bootstrap 样本 → 各自训练基学习器 → 平均预测
- **降低方差**（适合高方差模型如决策树）
- 基学习器可**并行训练**（embarrassingly parallel）
- 保证：集成 MSE ≤ 平均基学习器 MSE，但**不保证 ≤ 最好的**
- 多样性是关键——基学习器越不同，集成效果越好
- Out-of-bag 样本可用于内部评估

#### 3.3.3.2 Boosting

- 顺序构建，每次聚焦上一轮的错误样本
- AdaBoost.M1 核心步骤：
  1. 初始化等权重 wᵢ = 1/n
  2. 训练弱分类器 → 计算误差率 ηⱼ
  3. 计算 αⱼ = log[(1-ηⱼ)/ηⱼ]
  4. 错误样本权重 × exp(αⱼ)
- **降低偏差**（适合高偏差模型如浅树）
- 回归场景：替换指数损失为平方/绝对/Huber 损失
- XGBoost 就是 boosting 的现代实现

#### 3.3.3.3 Stacking

- 两层结构：基学习器(Level 0) + 超级学习器(Level 1)
- 超级学习器通过**交叉验证**训练（不需要额外 hold-out）
- 常用正则化回归作为超级学习器
- 区别于简单后处理：stacking 不需要两阶段数据划分

## 3.4 深思 (After Thought)

- 统计集成 vs 集成学习的核心区别：是否能访问**组件预测**
  - 统计集成：外部组合，可以看到每个组件
  - 集成学习工具箱：通常只输出最终组合结果
- 概率→确定性的转换**不简单**：
  - 用均值还是中位数？影响评估指标选择
  - 如果用均值做确定性预测，应该用 MSE 评估
  - 如果用中位数做确定性预测，应该用 MAE 评估
  - 大多数太阳预测者没有问过这些问题
