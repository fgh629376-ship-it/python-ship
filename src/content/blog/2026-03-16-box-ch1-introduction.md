# Box 时间序列分析 Ch1：五大应用领域与 ARIMA 框架的起源

> **教材**：George E.P. Box, Gwilym M. Jenkins, Gregory C. Reinsel — *Time Series Analysis: Forecasting and Control*, 4th Edition (Wiley, 2008)

## 为什么读这本书？

Box-Jenkins 方法论是时间序列分析的基石。无论你做金融预测、工业过程控制、还是光伏功率预测——ARIMA、转换函数、干扰模型这些工具都绑定在 Box 的名字上。这不是一本"介绍 ARIMA 的教程"，而是**发明 ARIMA 的人告诉你他为什么要发明它**。

第1章用 18 页篇幅回答了一个核心问题：**时间序列分析要解决什么问题？**

---

## 1.1 五大应用领域

Box 没有一上来就推公式，而是从五个实际问题出发，每个问题对应一类数学模型。这种"问题驱动"的写法是这本书最大的特色。

### 1.1.1 时间序列预测（Forecasting）

**问题**：已知 $z_t, z_{t-1}, z_{t-2}, \ldots$，预测未来 $z_{t+l}$，其中 $l$ 是前置期（lead time）。

**关键概念**：
- **预测函数** $\hat{z}_t(l)$：在时刻 $t$、前置期 $l$ 下的最优预测
- **目标**：最小化均方误差 $E[(z_{t+l} - \hat{z}_t(l))^2]$
- **概率置信限**：预测不是一个点，而是一个区间（50%、95% 概率带）

**对光伏预测的启示**：光伏功率预测本质就是这个问题——已知历史辐照度/功率序列，预测未来 1h、24h、72h 的输出。Box 告诉我们，最优预测的推导**必须基于一个已知形式的随机模型**，这就是为什么模型识别（Ch6）、参数估计（Ch7）、诊断检验（Ch8）构成了 Box-Jenkins 方法论的三步循环。

### 1.1.2 转换函数估计（Transfer Function）

**问题**：输入 $X_t$ 和输出 $Y_t$ 之间存在动态线性关系 $Y_t = \sum_{j=0}^{\infty} v_j X_{t-j}$，如何估计脉冲响应函数 $\{v_j\}$？

Box 用了一个化学炉的例子：输入是空气供给速率 $X_t$，输出是 CO₂ 浓度 $Y_t$，观测间隔 9 秒。关键难点是**噪声**——真实系统中，输入-输出关系总是被不可控扰动掩盖。

**对光伏的启示**：辐照度 → 功率就是一个典型的转换函数问题。NWP 预报 → 实际辐照度也是。转换函数模型允许我们量化"输入变化后多久、多大程度影响输出"——这在光伏系统中就是辐照度变化后功率响应的延迟和衰减特性。

### 1.1.3 异常干扰事件分析（Intervention Analysis）

**问题**：已知某个外部事件（政策变化、设备故障、特殊天气）在某时刻发生，如何量化它对时间序列的影响？

Box 和 Tiao (1975) 用干扰模型研究了洛杉矶空气污染控制法规对臭氧水平的影响。输入是一个只取 0/1 的指示变量。

**对光伏的启示**：
- 逆变器故障、面板清洗、雪覆盖 → 干扰事件
- 电网政策变化（限电、调度）→ 干扰事件
- Ch13 的干扰分析可以帮我们**自动检测和校正**这些异常值，避免它们污染模型参数估计

### 1.1.4 多元时间序列分析（Multivariate）

**问题**：$k$ 个相关时间序列 $\mathbf{Z}_t = (z_{1t}, z_{2t}, \ldots, z_{kt})'$ 的联合建模。

两个目的：
1. 理解多个序列之间的**动态关系**
2. 利用相关序列的信息**提高预测精度**

**对光伏的启示**：光伏系统天然是多变量的——GHI、DNI、DHI、温度、风速、湿度、功率输出——它们之间存在复杂的动态耦合。Ch14 的 VAR 模型可以同时建模所有变量，捕捉它们之间的 Granger 因果关系。

### 1.1.5 离散控制系统（Discrete Control）

**问题**：通过调整输入变量 $X_t$，使输出 $Y_t$ 尽量接近目标值 $T$。

Box 区分了两种控制哲学：
- **SPC（统计过程控制）**：Shewhart 控制图、CUSUM 图——被动监控，发现异常
- **EPC（工程过程控制）**：反馈/前馈控制——主动调整，补偿扰动

关键洞察：**预测和控制问题紧密相连**。最小均方误差预测直接导出最小均方误差控制。

**对光伏的启示**：光伏电站的无功补偿、储能系统充放电策略、逆变器 MPPT 控制——都是 EPC 问题。Ch15 的控制理论与预测理论的统一框架，是理解"预测驱动调度"的数学基础。

## 1.2 随机模型的数学框架

### 确定性 vs 随机性

Box 开篇就做了一个关键区分：
- **确定性模型**：物理定律精确计算（导弹轨迹）
- **随机模型（概率模型）**：计算未来值落在某区间的概率

**时间序列 ≠ 随机过程**。一个时间序列 $z_1, z_2, \ldots, z_N$ 是无穷随机过程的**一次样本实现**（sample realization）。统计推断的目标是从这一次实现推断总体性质。

### 线性滤波模型

这是整本书的数学起点。一个可观测的时间序列 $z_t$ 可以看作白噪声 $a_t$（独立同分布，均值 0，方差 $\sigma_a^2$）经过线性滤波器的输出：

$$z_t = \mu + a_t + \psi_1 a_{t-1} + \psi_2 a_{t-2} + \cdots = \mu + \psi(B) a_t$$

其中 $\psi(B) = 1 + \psi_1 B + \psi_2 B^2 + \cdots$ 是转换函数，$B$ 是后移算子（$Bz_t = z_{t-1}$）。

**物理直觉**：时序中观测值之间的依赖性，是由独立随机冲击（shocks）经过系统的"记忆"（权重 $\psi_j$）累积产生的。

**平稳条件**：若 $\sum_{j=0}^{\infty} |\psi_j| < \infty$，滤波器稳定，过程平稳，$\mu$ 是过程均值。

### 三大基本模型

#### AR(p) — 自回归模型

$$\tilde{z}_t = \phi_1 \tilde{z}_{t-1} + \phi_2 \tilde{z}_{t-2} + \cdots + \phi_p \tilde{z}_{t-p} + a_t$$

用后移算子：$\phi(B)\tilde{z}_t = a_t$

- $p+2$ 个未知参数：$\mu, \phi_1, \ldots, \phi_p, \sigma_a^2$
- **平稳条件**：$\phi(B) = 0$ 的所有根在单位圆外
- AR 是线性滤波的特例：$\tilde{z}_t = \phi^{-1}(B)a_t = \psi(B)a_t$

**AR(1) 的展开**：$\tilde{z}_t = \phi \tilde{z}_{t-1} + a_t$ → 递推代入 → $\tilde{z}_t = \sum_{j=0}^{\infty} \phi^j a_{t-j}$（当 $|\phi| < 1$ 时收敛）。这是理解"AR 可以表示为无穷 MA"的关键。

#### MA(q) — 移动平均模型

$$\tilde{z}_t = a_t - \theta_1 a_{t-1} - \theta_2 a_{t-2} - \cdots - \theta_q a_{t-q} = \theta(B) a_t$$

- $q+2$ 个参数
- "移动平均"这个名字有误导性——权重不需要为正，也不需要和为 1

#### ARMA(p,q) — 混合模型

$$\phi(B)\tilde{z}_t = \theta(B)a_t$$

- $p+q+2$ 个参数
- 等价线性滤波形式：$\tilde{z}_t = \psi(B)a_t$，其中 $\psi(B) = \phi^{-1}(B)\theta(B)$
- 实践中 $p, q$ 通常 $\leq 2$

### 非平稳扩展：ARIMA(p,d,q)

很多实际序列（股价、销量）不围绕固定均值波动。Box 的关键洞察：**差分可以将非平稳序列转化为平稳序列**。

如果广义自回归算子 $\varphi(B)$ 有 $d$ 个单位根：

$$\varphi(B) = \phi(B)(1-B)^d$$

定义 $w_t = \nabla^d z_t = (1-B)^d z_t$，则：

$$\phi(B) w_t = \theta(B) a_t$$

这就是 ARIMA(p,d,q) 模型。$d$ 通常取 0、1、最多 2。

**"积分"的含义**：ARIMA 中的 "I" 代表 Integrated（积分/求和），因为 $z_t = S^d w_t$，其中 $S = \nabla^{-1} = (1-B)^{-1} = 1 + B + B^2 + \cdots$ 是求和算子。平稳 ARMA 过程 $w_t$ 经过 $d$ 次累积求和得到非平稳 ARIMA 过程 $z_t$。

### 转换函数模型（带噪声）

输入-输出关系加上噪声：

$$Y_t = v(B)X_t + N_t = \delta^{-1}(B)\omega(B)X_t + \varphi^{-1}(B)\theta(B)a_t$$

需要同时估计：
1. 动态关系的转换函数 $v(B)$
2. 噪声模型的转换函数 $\psi(B)$

### 控制系统模型

误差 $\varepsilon_t = Y_t - T = v(B)X_t + N_t - T$

控制方程：$x_t = \zeta_1 x_{t-1} + \zeta_2 x_{t-2} + \cdots + \chi_0 \varepsilon_t + \chi_1 \varepsilon_{t-1} + \cdots$

**预测-控制统一**：最小 MSE 预测 → 最小 MSE 控制。

## 1.3 模型构建的基本思想

### 简约原则（Parsimony）

> "用尽可能少的参数获取充分的表示"

Box 举例：一个本该用 2 参数 $(1-\delta B)Y_t = \omega_0 X_t$ 表示的系统，如果用 $s+1$ 参数的多项式去拟合，不仅浪费数据信息，还会导致估计精度恶化。

**对光伏的启示**：这是过拟合警告。深度学习时代特别容易忽视简约原则——100 层 Transformer 可能不如 ARIMA(1,1,1) 在有限数据上表现好。

### 迭代建模三步法

这是著名的 Box-Jenkins 方法论核心：

1. **识别（Identification）**：用 ACF/PACF 等工具初步确定模型结构（$p, d, q$）
2. **估计（Estimation）**：最大似然法或最小二乘法估计参数
3. **诊断检验（Diagnostic Checking）**：残差分析，检查模型是否充分

不合格就回到第 1 步，如此循环直到找到满意的模型。

Box 特别强调：至少需要 **50 个观测值**，最好 **100 个以上**。

## 深度思考：Ch1 对光伏功率预测的启示

### 1. 五大领域在光伏中的统一

光伏功率预测其实同时涉及 Box 的全部五大应用：

| Box 应用领域 | 光伏对应场景 |
|-------------|-------------|
| 预测 | 1h/24h/72h 功率预测 |
| 转换函数 | 辐照度 → 功率的动态响应 |
| 干扰分析 | 设备故障、清洗、雪覆盖检测 |
| 多元时序 | GHI+温度+风速+湿度联合建模 |
| 控制系统 | 储能充放电调度、MPPT 控制 |

这意味着 Box 的框架不仅仅是"学一个 ARIMA"，而是提供了**从数据到决策的完整链路**。

### 2. 随机模型 vs 物理模型

Box 区分了确定性模型和随机模型。在光伏领域：
- **物理模型**（pvlib）：基于辐射传输方程，确定性
- **统计模型**（ARIMA）：基于历史数据，随机性
- **最佳实践**：物理模型给基线，统计模型捕捉残差——这就是"物理+统计"混合预测框架的理论基础

### 3. 简约原则的现代价值

2026 年深度学习盛行的背景下，重读 Box 1970 年提出的简约原则格外有启发：
- 光伏数据通常有限（一个电站几年的历史 ≈ 几千个日样本）
- 参数过多的模型在分布漂移时更脆弱
- ARIMA 的 3-5 个参数 vs Transformer 的百万参数——在小数据场景下前者可能更可靠

### 4. 迭代建模的工程意义

Box-Jenkins 三步法不只是学术方法论，它是一种**工程思维**：
- 不要一次就想找到完美模型
- 快速迭代：识别 → 估计 → 检验 → 修正
- 模型是工具，不是教条——如果 ARIMA(1,1,1) 残差通过检验，就不需要 ARIMA(3,2,3)

## 小结

Ch1 是一张路线图。Box 用 18 页告诉你：
1. 时间序列分析要解决五类实际问题
2. 所有这些问题都可以用**线性滤波器 + 白噪声**的框架来建模
3. ARIMA 是处理平稳和非平稳序列的统一工具
4. 模型构建是迭代的，简约的
